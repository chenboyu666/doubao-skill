# 来源核验(差异化护城河)

每份输出都要让用户看清信息从哪来、可信几分,但不要把内部风控标签直接堆到前台。

## 四分区

| 分区 | 含义 | 处理 |
|---|---|---|
| 官方 official | 公告、交易所、监管、央行 | 可直接采信 |
| 媒体转述 media | 主流财经媒体 | 关键事实宜 ≥2 源印证 |
| 机构观点 institution | 券商研报/机构判断 | 区分事实与观点 |
| 内部剔除 internal_excluded | 传闻、社区小作文 | 默认不进前台（规则见 `compliance.md`） |

## 一手来源优先级

宏观、政策、跨市场问题的核心证据不能主要靠低可信转引站点:

- 央行与利率:Fed、FOMC、ECB、PBOC、CME FedWatch 等
- 通胀与就业:BLS、BEA、统计局、财政部
- 能源与大宗:EIA、IEA、OPEC、交易所官方数据
- A 股盘面:交易所、指数公司、权威行情源、主流财经媒体复盘

「信息来源」模块前 5 条优先放一手/权威来源;其余为补充来源。

## 多源印证

- 多独立来源印证 → 正文可写「多源印证」
- 单一来源 → 弱提示「单一来源」,不抢占主信息
- 互相转载不算多源

## 前台来源露出

- 开篇写 `信源范围：` 一行说明
- 引用编号与链接规则见 `chat_contract.md`「信息来源」
- 来源名写用户可读名称：「上交所」「证券时报」,不写泛化「媒体」

## 行情与数据类来源（硬规则）

数据库、终端、API 取到的数字**可以**用于写正文，但**只有**找到对应**可打开网页**时才分配 `[n]` 并进入「信息来源」。

| 场景 | 检索/取数 | 进入「信息来源」 |
|------|-----------|------------------|
| 指数收盘、成交额、板块涨跌 | 可用东方财富、同花顺等 C 级站点取数 | **必须**改引一篇报道该数据的 B 级媒体稿或 A 级交易所/指数公司页面，URL 指向含该数字的文章 |
| PMI、CPI、就业等宏观 | 统计局、央行、BLS 等官网 | 用**发布稿原文 URL**，标题写发布稿题名 |
| 美股/港股指数 | 可用行情站取数 | 用主流财经媒体「收盘/隔夜」稿 URL，或指数编制方官方页 |
| 仅有终端/库内数字、搜不到文章 | 可写入正文 | **不得**编号；记 `coverage_gaps`，必要时写「该口径暂无公开原文链接」 |

**禁止的前台写法（一律视为交付失败）：**

- `…… · （行情数据）`
- `…… · 同花顺金融数据库 · …… · （行情数据）`
- 来源名含「数据库」「终端」「实时行情」且第四段不是 `https://` 链接
- 为凑编号把站点首页当 URL（如 `https://www.10jqka.com.cn/`）

**合并原则：** 同一交易日、同一指数的涨跌，若已在某篇媒体稿 `[n]` 中可核验，全文只保留该 `[n]`，不另开数据库条目。

## 来源去重

- 同一网页只有一个来源编号 `[n]`
- URL 规范化：忽略 utm、fragment、大小写、www、末尾斜杠
- 多网页印证同一事实 → 多个独立 `[n]`

## 内部工作笔记（不交付用户）

取数时可维护：

- `query_log`：检索词与意图
- `evidence_atoms`：claim / source / url / date / confidence
- `coverage_gaps`：缺口与建议检索词
- `conflicts`：冲突事实与处理方式

## 红线

- 媒体转述不能替代公告
- 传闻、社区小作文见 `compliance.md`「传闻纳入规则」
- 不把图表当证据入口；图表信息用文字写入正文
- 禁止占位 URL 与无链接却进「信息来源」的条目
- **禁止** `（行情数据）` / `（数据库）` 等假第四段；无 `https://` 文章链接则**无 `[n]`**
