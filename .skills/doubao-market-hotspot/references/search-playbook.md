# 检索剧本(search-playbook)

整个 Skill 的价值取决于这一步的质量:**搜得准、去得净、排得稳**。

## 工具无关原则

凡需联网,用宿主提供的搜索/抓取能力。**不要**写死工具名,**不要**用 curl/python 绕过宿主。搜不到 → 走兜底,不编造。

## 定向信源

| 级别 | 信源(站点) | 用途 |
|---|---|---|
| A 官方 | 巨潮、上交所、深交所、证监会、央行、公司公告 | 政策原文、确定性事实 |
| B 主流财经媒体 | 财联社、证券时报、上证报、中证报、第一财经、华尔街见闻 | 新闻、解读 |
| C 行情/数据/研报 | 东方财富、同花顺、雪球(研报摘要) | 数据、摘要(标口径) |
| D 社区/情绪 | 股吧、雪球讨论 | 仅内部情绪参考；默认不进正文（见 `compliance.md`「传闻纳入规则」） |

**C 级站点取数 ≠ 可编号来源。** 从 C 级拿到指数/成交额后，须再检索一篇 B 级媒体稿或 A 级官方页**报道同一数据**，用该文章 URL 作 `[n]`；细则见 `source-verification.md`「行情与数据类来源」。**禁止**输出 `同花顺金融数据库 · … · （行情数据）`。

海外事件取官方与一线外媒,映射到全球资产或 A 股**环节/方向**(不点名个股，见 `compliance.md`「脱敏规则」)。

转载、传闻、社区小作文在去重阶段按 `compliance.md`「传闻纳入规则」处理，默认丢弃。

## 检索角度

按 `question-types.md` 问题类型展开,每角度 2–3 个中文 query 变体。只做市场级检索,不展开成单股/板块深度。

## 检索问句

按 `analysis-playbook.md`「检索阶段」五问展开检索；每问落点见该表。

## Lens 检索补充

见 `analysis-playbook.md`「六个基础 Lens」；按主 Lens 追加检索角度(归因/影响/情景/定价/类比/主线)。

## 政策/宏观央行题

- 核心来源:央行官网、FOMC/ECB 官方页、FedWatch 等；媒体只作市场反应
- 任职人、会议日期、利率区间必须用官方来源核对,不用模型记忆
- 市场概率必须带 `as_of`,标明是「市场定价」不是央行承诺
- 「会不会跟随」类问题拆成:机械跟随可能、基准情景、什么数据会改变判断

## 内部审计字段（不交付）

见 `source-verification.md`「内部工作笔记」。

## 图片

关键增量区不配图；图表信息用文字写入正文或机制说明。

## 发布时间

- 每条尽量有可信发布日期
- 取不到日期 → 降权,标「时间存疑」
- 超出时间窗剔除

## 去重与合并

- 同事件多源 → 合并一条,最权威来源作主引用
- 同一网页 → 一个来源编号（规则见 `source-verification.md`「来源去重」）
- 转载、传闻 → 按 `compliance.md`「传闻纳入规则」处理

## 排序

按相关性、时效、来源可信度、市场影响、新颖度定档(高/中/低),不写数值分。默认取 Top 5–8 条进入正文。

## 检索预算

单次建议:搜索 8–12 次,深度抓取 6–8 篇。先广度各角度 1 次,再对 2–3 条核心事实加深。达上限即停,缺口在正文说明。

## 兜底

- 某角度无命中 → 标「该方向暂无可靠公开信息」
- 整体不足 → 「暂无足够公开信息支撑」+ 建议检索词
