# 第一阶段正文合同（写正文前读取）

本文件规定对话框正文的结构、深度、格式与写作风格。写正文前必须同时读取 `chat_example.md`：本文定结构与硬约束，范例定深度与因果链条长度。**A 股盘中归因**场景另读 `chat_example_market_intraday.md`。

## 输出顺序

1. 固定风险提醒（无标题，第一行）
2. **首屏**（合并原开篇 + 直接回答；无字段标签、无 `## 📌 直接回答`；含 `关键数据` 表，见下）
3. `## 📈 有哪些关键增量`
4. `## 🔍 为什么现在重要`
5. `## 🔗 如何影响市场`
6. `## 💬 各方观点与分歧`
7. `## 👀 接下来盯什么`
8. `## ⚠️ 风险提示`
9. `## 📚 信息来源`

正文最后逐字输出：

> 下一步是否为您生成飞书文档版？

收到肯定性回复即进入第二阶段。

## 交付终止（P0）

固定结尾句是用户可见输出的**最后一行**。写出该句后**立即停止生成**，不得在其后追加任何字符（含空行后的新段落）。

### 禁止在固定结尾之后出现

- 任务完成式元叙述：如「已按照…Skill 完成…分析」「我已完成市场热点分析」
- 正文 bullet 摘要 / TL;DR /「核心结论如下」及任何压缩版复述
- 改写或重复的飞书 CTA：如「你可以告知我是否需要生成飞书文档版」「需要的话我可以整理成飞书文档」
- 暴露 Skill 名、内部文件名、阶段名（「第一阶段」「对话框正文」「九模块」）
- 「以上」「综上」类收束语后再起新段

### 反例（不得输出）

```text
下一步是否为您生成飞书文档版？

已按照你提供的市场热点分析 Skill 完成豆包任务模式对光伏板块的影响分析，核心结论如下：
- …
你可以告知我是否需要生成飞书文档版的完整分析。
```

上例中固定结尾之后的全部内容为**违规追加**；飞书意向已在固定结尾问清，**不得**再问第二遍。

### 与首屏的关系

首屏 headline / 导语 / 作答+条件段已是可读结论层；模块 3–8 是展开证据。**禁止**在九模块之后再写第三层「结论摘要」——那会与固定结尾前的完整正文重复，并诱发二次 CTA。

## 固定风险提醒

第一行必须逐字输出：

`回答基于AI 生成，仅用于信息参考与研究辅助，不构成任何投资建议。股市有风险，请结合自身风险承受能力决策。`

- 不加标题、不加引用符号、不改写、不拆行。
- 生成飞书 XML 时不得搬运这行。

## 首屏（合并原开篇 + 直接回答）

风险提醒之后、第一个 `## 📈` 之前，**连续输出**下列内容。首屏**禁止**出现字段标签（`分析对象：`、`标题：`、`信源范围：`、`**关键数据：**`、`**核心观点摘要：**`、`**问题**：`、`**结论**：`、`**✅ 改善**`、`**❌ 恶化**`）及 `## 📌 直接回答` 标题；**禁止**引用块 `>`。

### 段落顺序

| 顺序 | 内容 | 句数 |
|------|------|------|
| 1 | 单独一行 **headline**（一句话核心判断，整行加粗；句首固定 `📌`，句末不加句号） | 1 行 |
| 2 | **导语**：市场范围与时间窗自然写入句首；解释 headline 为何成立；嵌入 2–3 个关键数字并带 `[n]`；盘中报告末句加数据局限；**事实句，不加粗** | ≤2 句 |
| 3 | **作答+条件**（**同一段**，不得空行拆开）：**作答**直接回答用户问题，**至少 1 处**关键分句加粗（见「加粗规则」；不必整句加粗）；紧接写 **条件** `若…则…；若…则需…`（**1 句**，含量化阈值，**常规字重不加粗**）；作答与条件句之间**不空行**；**不重复**导语已写的同一数字 | 作答 ≤2 句 + 条件 1 句 |
| 4 | **2×2 关键数据表**（见下） | — |

首屏与 `## 📈 有哪些关键增量` 之间必须有一行 `---`。首屏 **headline / 导语 / 作答+条件 / 表** 之间空一行；**作答+条件段内部不得空行**。

**飞书映射（组装时，用户正文不出现下列标签）：** 第一节「本段结论」取作答+条件段中作答部分前 1–2 句；「结论」取作答+条件段全文；「改善」「恶化」从同段条件半句拆分（见 `feishu-doc-style.md`）。

### 首屏版式（P1 · 块间空行必填）

首屏由 **五块** 组成：固定风险提醒 → headline → 导语 → 作答+条件（一块）→ 2×2 表。**块与块之间恰好空一行**；仅「作答+条件」块内部不空行。

**消歧：**「作答+条件同段、不得空行拆开」**只约束**作答句与条件句之间；**不取消**风险提醒 / headline / 导语 / 表 之间的空行。禁止把首屏整块写成无空行的连续长文。

### 关键数据表（首屏必填）

紧接作答+条件段之后、`---` 之前，输出 **2×2** Markdown 表格（**3–4 个指标**，末行可只填 1 格）。**不写** `**关键数据：**` 标题行。用于飞书 `SECTION:document-metrics`。

下列 fenced 示例中，**每个空行都是硬格式**（非渲染装饰）；写首屏时逐行对照，不得删空行、不得把多块合并成一段：

```markdown
回答基于AI 生成，仅用于信息参考与研究辅助，不构成任何投资建议。股市有风险，请结合自身风险承受能力决策。

**📌 点阵图偏鹰压制成长，但尚未触发全面 risk-off**

美联储 6 月 FOMC 决议发布至 6 月 30 日收盘：按兵不动符合预期，但点阵图下修 2026 年降息次数预期，10 年期美债收于 4.42%[2]。

短期偏成长承压、红利相对占优的风格切换，**不是全面熊市触发**；关键变量是 10 年期美债 4.5% 关口[1]——**若突破并伴随美元走强，北向与成长板块压力会明显加大。** 若美债收益率回落、北向连续净流入、成长相对红利收敛，则风格切换压力缓和；若通胀数据再超预期、点阵图进一步下修 2026 年降息次数、美元指数突破前高，则需警惕成长板块再度承压[1]。

| 指标 | 数值 | 指标 | 数值 |
|:----:|:-----:|:----:|:-----:|
| 10年期美债（收盘） | 4.42%[2] | 北向资金（当日） | 净流出 38 亿[3] |
| 创业板指（6月27日） | -1.1%[3] | 红利指数（6月27日） | 基本持平[3] |
```

要求：

- 分隔行固定为 `|:----:|:-----:|:----:|:-----:|`（**指标**列居中、**数值**列左对齐；见「视觉强调 · 表格列对齐」）。
- **指标**列写可读标签；盘中/发布后数据须在标签或数值旁标明口径（如 `沪指（9:41盘中）`、`PMI（9:30发布后）`），不得与前一交易日收盘混写而不标注。
- **数值**列写纯文本数字，**不加粗**；句末 `[n]` 必须来自模块 9（信息来源）且 URL 可打开（**禁止**数据库无链编号）。
- 优先选用户问题最关心的 **指数/成交额/核心宏观/外围锚**；归因题可增加 1 格「板块锚」（如 `医药生物（昨日）` +5.91%[5]）。
- 表中数字须与首屏导语/作答+条件段及后文一致；飞书阶段原句搬运，不得改写。
- 同一判断在 headline 与导语/作答+条件中**最多重复 2 次**；**不复述**用户原问题。

## 模块格式要求

每个主体模块（3–8）必须先写 **本段结论**（2–3 句），**空一行**后再展开细节。

**模块版式（P1 · 本段结论后必空一行）：** 本段结论与下一元素（事件卡 `### 📰`、四列表格、展开段、观点列表等）之间**必须有一空行**；不得 `**本段结论：……**` 紧接 `### 📰` 或表格首行。

```markdown
**本段结论：……（2–3 句）**

### 📰 6月28日｜信息来源：……｜……
**事实**：……
```

**本段结论格式（必填）：** 标签与正文同属一条加粗，写法为 `**本段结论：{2–3 句正文}**`（整段加粗）。**禁止**仅加粗 `本段结论：` 标签而让正文落在加粗外；本段结论块内**禁止**再嵌套句内分句加粗。

**本段结论内容（必填 · 模块级见解槽位）：** 2–3 句须覆盖三者——**综合判断**、**相对共识/预期的差**（预期差、真正变化、分歧焦点）、**对用户的启示**（意味着什么、盯什么变量）。

**展开段分工：** 本段结论之后写**证据、机制、细节、对比**（事实句不加粗）。若仍有不可替代的二阶判断，在句内用**关键分句加粗**标出最有价值的一个分句（0–1 处/模块）；可与前后句写在**同一段**内连续输出，**禁止**仅为加粗而空行拆段。

模块 4–8 本段结论与模块 3 相同：`**本段结论：{2–3 句正文}**` 整段加粗，**禁止**使用 `>` 引用块。

**模块标题（必填 emoji 前缀）**：

| 模块 | 标题 |
|------|------|
| 2 | （首屏，无 `##` 标题） |
| 3 | `## 📈 有哪些关键增量` |
| 4 | `## 🔍 为什么现在重要` |
| 5 | `## 🔗 如何影响市场` |
| 6 | `## 💬 各方观点与分歧` |
| 7 | `## 👀 接下来盯什么` |
| 8 | `## ⚠️ 风险提示` |
| 9 | `## 📚 信息来源` |

相邻 `##` 模块之间必须有一行 `---`（模块 9 与固定结尾问句之间不加）。

### 有哪些关键增量

3–5 条催化，按事件日倒序。**必须**使用事件卡片格式（见下「事件卡片」），以便 MD→XML 转换。卡片与卡片之间必须有一行 `---`：

```markdown
### 📰 {date}｜信息来源：{source_name}｜{title}
**事实**：……（纯文本数字，不加粗）
**机制**：……
**市场反应**：……（定价层判断须用**关键分句**加粗标出句内最有价值的 1 个分句，可与事实句写在同字段、同段，见下）
**后续验证**：……

---

### 📰 {date}｜……
```

禁止使用紧凑列表或其它变体。

### 为什么现在重要

写预期差、真正变化、共识与分歧；不要复述关键增量标题。

### 如何影响市场

**必须**使用四列传导表（见下「四列传导表」）；模块 5 **禁止**用纯段落机制说明替代表格，以便 MD→XML 稳定转换。

### 各方观点与分歧

按角色组织，每条 1–2 句 + 句末 `[n]`，角色名带固定 emoji：

```markdown
- **📋 官方事实**：……[1]
- **📊 市场定价**：……[2]
- **🏦 机构观点**：……[3]
- **🔄 反向证据**：……[2]
```

### 接下来盯什么

3–4 条信号，固定格式（emoji 必填）：

```markdown
**信号1 · 📌 短标签**：具体被观察对象
**👀 盯**：观察口径；**必须含可量化阈值**（如「收盘 ≥3 万亿」「连续 3 个交易日」「有效突破 4100 点」）
**✅ 改善**：阈值满足后对主结论的含义
**❌ 恶化**：阈值恶化后对主结论的含义；**必须含量化数字或百分比门槛**（纯文本，不加粗）

---

**信号2 · 📌 ……**
```

信号条之间用 `---` 分隔。

**阈值硬要求（P1）：**

- `👀 盯` 行不得只有「放量」「走弱」「关注持续性」等模糊词；须写清观察什么 + 多少算满足/不满足。
- `❌ 恶化` 行须写清量化触发（如成交额 ≤2.5 万亿、板块单日 ≥5 只跌停、回调 ≥10%）。
- 盘中报告：优先写 **收盘后可验证** 的信号；若只能盘中观察，须在 `👀 盯` 标明「盘中暂态」并降低结论强度。

### 风险提示

3–5 条，固定格式（emoji 必填）：

```markdown
**⚠️ 风险1 · 风险标题**
**🔴 触发**：可观察条件 + 为何威胁主结论（同段写完）
**🟢 证伪**：什么出现会削弱该风险或推翻主结论

---

**⚠️ 风险2 · ……**
```

风险条之间用 `---` 分隔。

### 信息来源

列出正文实际引用的全部来源，每行格式（**四段缺一不可**）：

`[n] 来源名 · 日期 · 标题 · URL`

**硬门禁：无 URL = 无编号**

- 第四段 `URL` **必须**以 `http://` 或 `https://` 开头，且为真实可打开的网页链接。
- **禁止**用 `（行情数据）`、`（数据库）`、`（终端数据）`、`（实时行情）`、`无链接`、`N/A`、`-`、空字符串等任何非 URL 文本充当第四段。
- **禁止**写「同花顺金融数据库」「Wind」「彭博终端」等**无对应网页**的来源名并强行编号；这类数据只能在检索阶段使用，**不得**进入本列表。
- 正文里每一个 `[n]` 都必须在本列表有一行且带完整 URL；本列表每一行都必须被正文至少引用一次。
- 若某事实找不到可核验 URL：**不要**给它 `[n]`。改引带链接的媒体复盘、交易所/指数公司公告、官方统计页面；或在正文写清数字但**不带编号**（如「据公开收盘数据，沪深300……」），并在 `coverage_gaps` 记缺口（见 `source-verification.md`）。

**引用编号规则：**

- 正文句末只写 `[n]` **编号**，不写 Markdown 链接、不把 URL 嵌入句中。
- 可点击链接只在模块 9 本列表（及飞书第七节）；飞书正文中 `[n]` 保持纯文本。
- 无可靠公开 URL 的来源**不得**分配 `[n]`、不得进入本列表；可在模块7 写「据 XX 机构观点」但不带编号。
- 禁止占位 URL（如 `example.com`、首页泛链 `https://www.eastmoney.com/`、搜索页、仅域名无文章路径的链接）。

**行情/指数数字怎么编号（常见纠错）**

- 指数涨跌、成交额、北向净流入等盘面数字，**优先**引用**同一数据点**出现在正文里的那篇**媒体收盘综述/操盘必读**（如证券时报、新浪财经、东方财富财富号）的 URL，**不要**单独再列一条「XX 金融数据库 · 行情数据」。
- 若已有 `[5] 东方财富 · A 股下探回升` 含当日指数，正文指数应用 `[5]`，**不得**再开 `[4] 同花顺 · A 股指数实时行情 ·（行情数据）`。
- 宏观数据（PMI、CPI 等）用发布机构原文页（如国家统计局链接），不用数据库名顶替。

来源可信度、去重、行情取数与多源印证见 `source-verification.md`。

### 飞书准入门禁（P0）

满足以下**任一条**即**不得**进入第二阶段飞书 XML 组装；须先修正第一阶段正文：

1. 模块 9 任一条第四段不是 `http://` / `https://` 开头的文章 URL（含 `（行情数据）`、空链接、站点首页泛链）。
2. 正文出现 `[n]` 但模块 9 无对应行，或模块 9 有无被引用的孤儿编号。
3. 首屏 `关键数据` 2×2 表缺失，或表中任一格无 `[n]` / `[n]` 不可打开。
4. 首屏缺 headline、导语、作答+条件（同一段）或 2×2 表，或出现禁止字段标签。
5. 模块 3 事件卡 `市场反应` 未写定价层（见下「事件卡片 · 市场反应」）。
6. 模块 7 任一条信号 `👀 盯` 或 `❌ 恶化` 无量化阈值。

## 传导链格式（模块 3、5）

「如何影响市场」与事件卡片中的机制说明，统一按本节执行。

### 禁止

- 英文箭头 `→`、`↓` 及单独一行的箭头链
- emoji 作为传导链节点或句内装饰（如 `🟠政策🟠→🔴市场`）；**例外**见下「视觉强调」定点 emoji 表
- 纯标签式空话：「利好市场」「提振情绪」「短期催化」
- 模块 5 用纯段落机制说明替代四列表格

### 四列传导表（模块 5 必用）

对话框 Markdown 与飞书 XML 共用同一逻辑；飞书用 `feishu-doc-style.md` 固定 colgroup。

```markdown
**本段结论：……（2–3 句，整段加粗）**

| 环节 | 变化 | 传导机制 | 市场含义 |
|:----:|:-----|:---------|:---------|
| 政策事实 | 6月FOMC按兵不动，点阵图偏鹰[1] | 降息预期后移 | 流动性宽松预期降温 |
| 利率定价 | 2年期美债收益率上行约12bp[2] | 短端利率是估值折现锚 | 高估值成长承压 |
| 风格反应 | 成长指数跑输红利约1.2pct[3] | 资金向确定性迁移 | 风格切换而非全面 risk-off |
```

要求：

- 3–5 行；超过 5 行合并为「近端/远端」两层
- 分隔行固定为 `|:----:|:-----|:---------|:---------|`（**环节**列居中、后三列左对齐；见「视觉强调 · 表格列对齐」）
- 每行至少一个 `[n]`；机制列写因果，不只写形容词
- 表前必须有整段加粗的 `**本段结论：……**`，且与表格之间空一行
- 机制列与结论句中**不用** `→`

### 事件卡片（模块 3 必用）

```markdown
### 📰 6月28日｜信息来源：中国人民银行｜降准 0.5 个百分点

**事实**：……
**机制**：降准降低银行资金成本，信贷供给意愿上升，短端利率下行；若经济预期仍弱，效果可能更多体现在债券而非权益。
**市场反应**：消息公布后沪深300 30 分钟内上涨约 0.4%[2]。相对此前「仅托底、难宽信用」的共识，**短线定价偏利好兑现，尚未确认盈利预期上修。**
**后续验证**：盯 7 天内社融与 M1 是否同步改善；若信贷需求仍弱，则本次降准更像流动性托底而非盈利预期上修。

---
```

**机制**字段用中文分句写因果，不用 `→`。

#### 事件卡片 · 市场反应（P1 · Pricing 必填）

`市场反应` **不得**只复述价格涨跌或复制 `事实` 句。每条须用 **1–2 句**写清定价层（对应 `analysis-playbook.md` Pricing Lens），至少覆盖其一：

| 要素 | 写法提示 |
|------|----------|
| 相对预期 | 超预期 / 符合预期 / 低于预期 / 市场已提前交易 |
| 价格行为 | 发布后 N 分钟/当日涨跌幅[n]，或「未明显反应」 |
| 对冲因素 | 若被其它因素抵消，写明「被 X 对冲」（如半年末资金面、海外隔夜） |

**差例：**「有色金属板块大涨。」（仅复述结果）  
**合格例：**「数据公布后指数未明显拉升，说明 PMI 回暖已在预期内，且被半年末调仓抛压对冲[4]。」

### 与 Lens 的关系

传导链体现 Impact / Scenario / Pricing 等 Lens（见 `analysis-playbook.md`），但不写 Lens 名称。主 Lens 决定链条重心（归因题偏「为何反应」，政策题偏「变量到资产」）。

## 深度标尺

- 每个 `**本段结论：……**` 含：**综合判断** + **相对共识/预期的差** + **对用户的启示**；**整段加粗**
- 除 headline、本段结论外，段落正文用**关键分句**加粗（每句最多 1 处）；**禁止**整句加粗长复合句、**禁止**点状 `**词**`；表格单元格不加粗
- 写完后自检：扫读 headline + 各本段结论 + 展开段关键分句，能否在 30 秒内说出主线？
- 关键增量写事实，不写「利好/利空」标签式空话
- 每条事件卡 `市场反应` 写清 **定价层**（预期差 / 是否 price in / 对冲因素），定价判断以**关键分句**加粗（非整句）
- 传导链写机制，不只写结果；模块 5 表格与模块 3 机制**不逐条复述**（表格写跨事件汇总路径）
- 机制与信号用**中文因果句**，不用英文箭头 `→` 连接
- 模块 7 每条信号含可量化阈值（纯文本，不加粗）
- 首屏 `关键数据` 表与正文数字一致，且每条可溯源 `[n]`
- 至少 1 条反向信号、1 条对应风险直接回应用户问题

## 视觉强调（必填）

目标：读者扫**加粗分句**即可抓住每句最有价值的判断，同时保持段落连续可读；所有 `**…**` 须能映射到飞书 XML（见 `feishu-doc-style.md`）。

### 分隔线

- 首屏 `关键数据` 表之后、`## 📈 有哪些关键增量` 之前：`---`
- 每个 `##` 模块之前：`---`（首屏内部不加）
- 事件卡片之间、信号条之间、风险条之间：`---`
- **禁止**在表格行之间插入 `---`

### 引用块

对话框正文**禁止** `>` 引用块（含模块 4「为什么现在重要」的本段结论；与其它模块一致，仅整段加粗 `**本段结论：……**`）。首屏亦不用引用块。

### 段落间距（P1 · 硬格式）

**空行 ≠ 拆段偷懒：** 下列位置**必须**有空行。与「作答+条件同段」「禁止为加粗拆段」**不冲突**——后者仅约束**同一段内部**，**不取消**块与块之间的空行。

**仅以下位置禁止空行**（不得拆成两段）：

- 首屏「作答+条件」段内部（作答句与条件句之间）
- 事件卡片四字段（事实 / 机制 / 市场反应 / 后续验证）之间
- 信号条 / 风险条内部各行之间
- 各方观点列表各项之间

对话框正文中，**长段落**（≥2 句，或单句但 ≥60 字）之间必须空一行，包括但不限于：

- 固定风险提醒与 headline 之间
- 首屏 headline / 导语 / 作答+条件 / 表 之间（作答+条件为一块，内部不空行）
- 每个 `##` 模块内：`**本段结论：……**` 与后续首段、事件卡、表格、列表之间
- 同一模块内相邻长段落之间（如模块 4 本段结论后的各展开段之间）

#### 输出前空行扫描（必填）

写出固定结尾问句之前，自上而下检查：

1. 首屏五块（风险提醒 / headline / 导语 / 作答+条件 / 表）之间是否**各有一空行**
2. 每个模块 `**本段结论：……**` 与下一元素之间是否**有一空行**
3. 模块内若有 ≥2 个长展开段，段间是否**各有一空行**

任一处两块长段落直接相连（中间无空行）→ 视为**格式失败**，须补空行后再交付。

### 表格列对齐

对话框表格用 GFM 分隔行冒号语法控制列对齐（作用于**整列**，非仅表头）。飞书阶段按 `feishu-doc-style.md` 独立映射，分隔行对齐符不进入 XML。

| 表格 | 分隔行（固定） |
|------|----------------|
| 首屏 2×2 关键数据 | `\|:----:|:-----:\|:----:|:-----:\|` |
| 四列传导表 | `\|:----:|:-----\|:---------\|:---------\|` |

- **指标 / 环节**列（标签列）：`:----:` 居中，与飞书灰底标签列一致。
- **数值 / 变化 / 传导机制 / 市场含义**列：`:-----` 左对齐（长文本列可读性优先；表头因 GFM 限制同为左对齐，飞书文档侧表头仍居中）。
- **禁止**四列全 `:----:`（长文本列居中降低扫读效率）。

### 加粗规则（P0）

加粗分**两个层级**：

1. **整块加粗**：headline、`**本段结论：……**`（标签+正文整段包在一个 `**……**` 内）
2. **句内关键分句加粗**：段落与字段正文中的二阶判断，**只加粗句内最有价值的 1 个完整分句**——不是整句、不是词语

**关键分句**定义：以 `，` `；` `——` 或「并非/而是/意味着/关键在于/本质」等转折·结论标记为自然边界的一个**完整语义分句**；通常 ≥8 字，或虽短但是完整判断（如「并非 panic 式抛售」）。**每句最多 1 处**关键分句加粗。

**选哪一段加粗（写前必问）：** 同一句里若有铺垫 + 判断，加粗**判断/结论/定价层**分句，不加粗事实铺垫。常见选法：

| 句型 | 加粗位置 | 示例 |
|------|----------|------|
| 事实 + 结论 | 后半（结论分句） | `…黄白线大幅收敛，**显示赚钱效应从机构重仓股向全市场扩散。**` |
| 对照/转折 | 含「并非/而是」的分句 | `…风格切换，**不是全面熊市触发**；关键变量是…` |
| 预期差/定价 | 含预期差判断的分句 | `…指数未明显拉升，**说明 PMI 已在预期内，且被半年末抛压对冲。**` |
| 单短句且全是判断 | 可整句（≤35 字、无逗号分句） | `**期货定价与点阵图尚未 price in 完毕。**` |

### Markdown 语法（P0 · 硬门禁）

- `**` 与文字**必须紧贴**，两侧**不得**有空格
- ✅ `**本段结论：正文。**`、`**📌 标题**`、`**一个完整分句。**`
- ❌ `** 本段结论： **`、`** 完整一句。 **`

若 `**` 与文字间有空格，渲染器会把 `**` 当普通字符显示（前台出现裸 `**` 符号），视为**格式失败**，输出前必须修正。

正文 `**…**` **只允许四类**：

| 类型 | 格式 | 说明 |
|------|------|------|
| headline | `**📌 {一句话}**` 独立一行 | 全文总判断；句首固定 📌，句末不加句号 |
| 本段结论 | `**本段结论：{2–3 句正文}**` | 每 `##` 模块开篇；标签+正文**整段加粗** |
| 关键分句 | 句内 `…**{一个完整分句}**…` | 每句最多 1 处；`[n]` 在 `**` 外 |
| 格式标签 | `**事实**`、`**👀 盯**`、`**📋 官方事实**` 等 | **仅标签词**加粗，冒号后正文不加粗（正文内再用关键分句规则） |

**三层边界（写前必辨）：**

| 层级 | 做法 | 示例 |
|------|------|------|
| ✅ 本段结论整段加粗 | 模块 3–8 开篇；标签+2–3 句包进**一个** `**……**` | `**本段结论：句1。句2。句3。**` |
| ✅ 关键分句加粗 | 句内只包 1 个最有价值的分句 | `…共识，**短线定价偏 hawkish 兑现，尚未触发 panic。**` |
| ❌ 长句整句加粗 | 多子句全部包进 `**……**` | `**与前几周…不同，今日…收敛，显示…提升。**` |
| ❌ 点状加粗 | 只加粗词、术语或数字 | `市场按 **re-pricing** 定价`、`超 **3000** 只个股上涨` |

**段落规则（消歧）：** 「禁止空行」**仅适用于**上表四类内部；**不适用于**风险提醒、headline、导语、本段结论、模块块之间——那些位置**必须**空一行（见「段落间距」）。

- 首屏**作答+条件**写在**同一段**（句与句之间不空行）；作答内 **≥1 处**关键分句加粗，条件句常规字重
- 模块内关键分句可与前后普通句写在**同一段**连续输出；**禁止**仅为加粗而在关键分句前后插入空行
- **块间仍须空一行**：风险提醒 / headline / 导语 / 作答+条件 / 表；各模块本段结论与后续内容

### 关键分句（必填）

承载**二阶判断或前瞻**、去掉后读者会误判主线的**一个完整分句**（不是整句里的全部子句）。引用编号写在加粗外：`…**观点分句。**[3]`

**识别启发式（加粗的分句须满足其一）：**

| 信号 | 示例 |
|------|------|
| 对照结构 | 「并非…而是…」「不是…而是…」 |
| 二阶词 | 意味着、关键在于、真正变化、预期差、胜负手、本质 |
| 前瞻结构 | 若…则…、需警惕…、将升级为… |
| 定价层 | re-pricing 而非 panic、利好被对冲、price in 完毕 |

**排除（不得加粗）：** 以时间窗 + 指数/成交额数字为主的事实分句；纯机制铺垫分句；与本段结论同义的重复判断；**孤立词/缩写/裸数字**。

**配额：**

| 区域 | 关键分句 |
|------|----------|
| 首屏 headline | 1 行（整行，已有） |
| 首屏导语 | 0（事实为主） |
| 首屏作答+条件 | **同一段**；作答 **≥1 处**；条件 **1 句**常规字重 |
| 每模块本段结论 | **1 段**整段加粗（不计入配额） |
| 每模块展开段 | 0–1 处 |
| 事件卡 `市场反应` 定价层 | 1 处（可与事实句同段） |
| 全文关键分句合计（不含 headline、本段结论、格式标签） | **≤12 处** |

## 写作风格

目标：输出像中文财经媒体的简报，不是英文 dashboard 的直译。不暴露 Skill 名、内部文件、XML 组装过程。

**单轮单交付：** 第一阶段在**同一条回复**内输出风险提醒 → 九模块正文 → 固定结尾，然后停止。不得拆成「先给摘要、再问要不要全文」；不得在同轮回复末尾再附「核心结论如下」bullet 列表。

### 标点与空格

- 使用中文全角标点：`，` `。` `、` `：` `；` `（）` `「」`
- headline 句首固定 `📌`（写在加粗内），句末不加句号
- 中文与数字、英文缩写之间不加空格：近7天、24小时、A股
- 百分号、倍数、时间不加空格：15%、2倍、6月11日15:30
- **不使用英文箭头 `→` `↓`、圈号**作正文连接
- emoji **仅**允许出现在「视觉强调」定点槽位；不在因果句、机制句、表头中随意插入

### 日期与时间

- 前台日期：`6月11日` 或 `6月11日15:30`
- 时间范围：`6月4日—6月11日`
- 写明口径：收盘/盘中/事件发布日起
- **盘中报告**：首屏导语须标明数据截至时刻并加局限句；模块 7 优先写收盘后可验证信号

### 栏目命名

- 不使用 Headline、Highlights、Original 等英文栏目标
- 不在前台出现 unverified、source、corroboration 等内部词

### 句式

- 先说结论，再说原因；一句一意
- 关键增量写标题之外的依据、影响范围，不复述标题
- 多写「发生了什么」「影响什么」「后续验证什么」
- 能说「市场担心钱会流走」，不说「流动性约束边际强化」
- 避免字段名式表达；改成「确认主线」「推翻主线」「接下来盯什么」
- 连贯完整句，用「因为/但/目前来看/关键在于」连接因果
- 判断明确，语气是解释不是命令
- 正文不写「据报道」「某某来源公开信息显示」；来源在句末 `[n]` 与文末列表
- 专业词第一次出现顺手解释
