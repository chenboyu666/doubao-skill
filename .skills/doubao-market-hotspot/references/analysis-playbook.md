# 分析 Playbook（Lens + 洞见）

本文件合并市场分析视角（Lens）与洞见框架。Lens 只约束思考方式，不规定输出格式；最终简报按 `chat_contract.md` 写成一体化分析，不按 Lens 分章节，不暴露「我选择了哪些 Lens」。

## 一、Lens 使用原则

1. 先判断问题需要哪些 Lens，再检索和分析。
2. 每次选择一个主 Lens，最多选择 1–3 个辅助 Lens。主 Lens 回答用户的核心问题，辅助 Lens 补齐证据、边界和验证条件。
3. 一个问题不要强行归入单一模式。若用户同时问「为什么、影响、会不会、是否已反映」，组合多个 Lens。
4. Lens 影响取证角度、证据权重和叙事重心，不生成前台固定栏目。
5. 若证据不足，降低结论语气，写清缺口和后续验证信号。

## 二、六个基础 Lens

### Attribution（市场归因）

目标：解释市场为什么发生当前表现，识别真正驱动市场定价的核心原因，而不是简单罗列同时发生的新闻。

分析过程：先观察市场真正定价的对象（指数、风格、行业、资产、风险偏好）；再识别所有可能驱动因素，并判断哪些属于新增信息、哪些只是背景；按照对市场解释力排序，区分主因、次因和伴随因素；最后分析为什么市场选择在当前时点反应，以及哪些变化已经反映在价格中。

避免：不要把所有新闻都当成原因；不要看到新闻就默认是涨跌原因；不要忽略市场预期和仓位因素。

### Impact（市场影响）

目标：分析一个事件如何改变市场，而不是简单罗列受益行业。

分析过程：首先识别事件真正改变了哪个变量（政策、需求、成本、利率、风险偏好等）；然后分析变量如何逐层传导到资产、行业、企业和市场；区分哪些影响立即发生，哪些需要时间兑现；最后判断整个传导链中最值得关注的验证节点。

避免：不要直接给受益股名单；不要跳过中间传导过程；不要把所有影响都视为同等重要。

### Scenario（市场推演）

目标：分析未来最可能如何演化，而不是直接预测结果。

分析过程：先找出真正决定未来走势的关键变量；分析这些变量当前状态；构建几个主要演化路径及触发条件；判断目前最可能的路径，并明确哪些新增信息会改变当前判断。

避免：不要给确定性预测；不要只讨论一种可能；不要脱离当前变量空谈未来。

### Pricing（市场定价）

目标：分析市场如何理解这件事，而不仅仅分析事情本身。

分析过程：先还原市场此前的主流预期；判断本次信息相对于预期属于确认、超预期还是低于预期；分析哪些已经 Price In，哪些仍存在预期差；最后判断是否仍有重新定价空间。

避免：不要只分析事件事实；不要把事实变化和价格变化混为一谈；不要忽略市场预期已经提前交易的部分。

### Analogy（历史类比）

目标：利用历史帮助理解当前，而不是照搬历史结论。

分析过程：寻找最具有参考价值的历史案例；比较当时与当前的宏观环境、政策背景、估值水平、市场结构和资金环境；分析哪些经验仍然适用，哪些条件已经发生变化；最后判断历史案例能够提供哪些启发。

避免：不要因为表面相似就直接下结论；不要忽略当前环境已经变化。

### Regime（市场主线）

目标：识别市场真正交易的核心逻辑，而不是总结热点新闻。

分析过程：观察资金持续流向、行业扩散、风格变化和风险偏好；识别哪些主题正在形成市场共识，哪些只是短期催化；判断当前主线是在强化、衰减还是切换；分析未来最可能改变主线的触发因素。

避免：不要把板块轮动当作主线切换；不要简单按照涨幅罗列热点。

## 三、问题到 Lens 的 Playbook

### 问题类型 → Lens（与 `question-types.md` 编号对应）

| # | 类型 | 默认 Lens |
|---|---|---|
| 1 | 全市场热点 / 交易主线 | Regime + Pricing |
| 2 | 市场涨跌归因 | Attribution + Pricing |
| 3 | 宏观 / 政策事件影响 | Impact + Pricing + Scenario |
| 4 | 新闻 / 风险事件影响 | Impact + Scenario |
| 5 | 跨资产联动 | Impact + Pricing |
| 6 | 风险偏好 / 市场情绪 | Regime + Attribution + Pricing |
| 7 | 市场级更新追踪 | Pricing + Regime 或 Impact |

上表是入口识别后的默认起点，不是硬规则；可按用户问题微调主 Lens 与 1–3 个辅助 Lens。

### 按用户问题特征（补充）

问题类型未覆盖时，按特征选 Lens（完整类型与 few-shot 见 `question-types.md`）：

| 用户问题特征 | 默认 Lens |
|---|---|
| XX 会不会发生 | Scenario + Impact |
| XX 后续怎么看/还能涨吗 | Scenario + Pricing |
| 会不会像 XX | Analogy + Impact |
| 长鑫 IPO 会不会冲击市场 | Impact + Analogy + Pricing |
| 豆包办公模式对 A 股影响 | Impact + Pricing |

### Lens 组合写法

- Attribution + Pricing：先解释主要驱动，再说明这些驱动此前是否已被价格反映。
- Impact + Pricing：先讲事件改变的变量，再讲市场是否已经提前交易、还有没有预期差。
- Scenario + Impact：先给最可能路径和条件，再讲每条路径的市场传导。
- Scenario + Pricing：先讲当前价格隐含了什么，再讲哪些信号会打开新的上行或下行情景。
- Analogy + Impact：先说明历史案例的可比与不可比，再把可迁移经验转成当前影响链。
- Regime + Pricing：先判断市场主线是否强化或切换，再说明价格和资金是否已经拥挤。

### 内部 Lens 选择记录（不交付用户）

```json
{
  "primary_lens": "Attribution",
  "supporting_lenses": ["Pricing"],
  "why": "用户问今天为什么跌，需要先归因，再判断市场此前是否已预期这些因素。",
  "must_answer": ["主因/次因/噪音", "此前预期", "当前是否重新定价", "证伪信号"]
}
```

## 四、洞见框架

目标：避免简报沦为「信息罗列」。每份输出必须把公开信息转成**可验证的市场解读**，不提供买卖建议。

### 检索阶段（执行见 `search-playbook.md`）

| 问题 | 目的 | 落点 |
|---|---|---|
| 事实变化 | 本窗口真正新增了什么 | 关键增量 |
| 来源可信度 | 官方/媒体/机构怎么说 | 信息来源 |
| 市场含义 | 影响变量与环节 | 如何影响市场 |
| 反向证据 | 降温、证伪、澄清 | 各方观点 / 风险 |
| 后续验证 | 接下来观察什么 | 接下来盯什么 |

搜不到可靠信息时记入内部 `coverage_gaps`（字段定义见 `source-verification.md`），正文说明缺口，不硬编。

### 写作阶段（内部分析用，不暴露前台）

1. 一句话判断：这批信息合起来说明了什么？
2. 为什么现在重要：新政策、新预期修正，还是过时信息重炒？
3. 真正变化：相比上一阶段，事实、预期或节奏哪里变了？
4. 各方观点：官方、市场定价、机构、反向证据分别说什么？
5. 正向条件：哪些事实会让核心观点更站得住？
6. 反向条件：哪些事实会削弱或推翻核心观点？
7. 验证/证伪信号：后续盯什么来确认或推翻主线？

### 内部分析与对话框模块

| 内部分析 | 对话框模块 |
|----------|------------|
| 一句话判断 + 写作问句综合 | 开篇 + 直接回答 |
| 关键事实 | 有哪些关键增量 |
| 为什么重要 + 真正变化 | 为什么现在重要 |
| 传导链 | 如何影响市场 |
| 各方观点 | 各方观点与分歧 |
| 验证/证伪 | 接下来盯什么 |
| 来源账本 | 信息来源 |

输出格式见 `chat_contract.md`；飞书 MD→XML 映射见 `feishu-doc-style.md`。

### 信息颗粒度

- 关键增量只写事实摘要；解释性话术进入「为什么重要」「如何影响市场」
- 传导链每步写清：这一步为何成立、通过什么机制、对市场定价意味着什么
- 各方观点每条回答：谁怎么看、为什么这么看、分歧在哪里
- `**本段结论：……**` 整段加粗，不是栏目摘要，而是结合主 Lens 的二阶判断
- 除本段结论外，二阶判断用**关键分句**加粗（每句最多 1 处）；**禁止**长复合句整句加粗、**禁止**词/数字点状加粗；**禁止** `**` 与文字间有空格；事实句与表内数字不加粗（见 `chat_contract.md`「加粗规则」）

### 合规边界

红线、脱敏、传闻与免责声明见 `compliance.md`。本文件只补充分析质量要求：

- 允许：市场方向、受影响环节、节奏、预期差、验证路径
- 证据不足：写「信心低」和缺口，不硬编

### 反信息罗列检查

- 去掉核心判断后仍像新闻列表 → 洞见不合格
- 正向与反向条件都能成立 → 必须给出验证/证伪信号
- 观点去掉来源后仍成立 → 太泛，需回到证据重写
