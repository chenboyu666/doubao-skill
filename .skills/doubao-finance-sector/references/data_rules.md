# 数据真实性：分级与硬规则

> 长参考文件：默认先读 `references/data_contract.md`。只有分级争议、聚合页边界、二级/三级来源判断不清楚时，再读本文。

板块热度分析是金融交付，读者会拿每个数字、每条事实去做投资决策，**数据真实性优先于排版、风格与篇幅**；数据分级原则以本文件为准，取数流程以 `references/data_contract.md` 为准，字段合同与文档 payload 校验以 `references/payload_fields.md` / `scripts/validate_doc_payload.py` 为准，取数笔记与缺口记录以 `references/data_collection_deep_dive.md` 为准。

## 目录

- 第一部分 · 三级可信度分级
- 分级例子与措辞速查
- 第二部分 · 七条硬规则

本文件分两部分：**第一部分**定义三级可信度分级（决定每条数据能不能作展示值、措辞要多克制）；**第二部分**是把分级与口径落到操作上的七条硬规则。两部分配套使用——第二部分的规则 4 / 5 / 6 正是第一部分分级的执行细则。分级结果连同来源与建议措辞，逐条登记到 facts.json（见 `references/facts_schema.md`）。

## 第一部分 · 三级可信度分级

进入分析的每条数据，先归入下面三级之一。

### 第一级 · 可直接作为事实陈述（最高可信）

两类来源同属一级：

1. **金融数据库（`seed_finance_search`）的行情 / 财务数据**——板块与个股的价格、涨跌幅、成交额 / 换手 / 量比、个股 / 代表股估值（PE / PB / PS）、区间高低点、涨停 / 上涨家数等。**这是一切行情数字的唯一来源。固定模块禁止采集、回填或展示板块 PE / 板块市盈率；估值只用选定 10 只代表股的股票 PE(TTM)。**
2. **一手官方数据与权威机构数据**（多经 `general_search` 触达原文）——按维度举例：
   - **政策 / 法律**：中国政府网、全国人大网的政策与法律原文；
   - **监管 / 交易所动作**：证监会、沪深北交易所、央行、国家统计局的公告、规则、数据；
   - **公司经营 / 披露**：公司年度报告、公司季度报告、业绩快报、公司公告（巨潮资讯 / 交易所披露）、投资者关系活动记录表；
   - **宏观数据**：国家统计局、央行、海关总署等的官方数据发布；
   - **官方通讯社**：新华社等官方媒体的原发稿；
   - **机构研究**：券商研报原文（署名、带日期）。

一级数据可直接写成事实（"披露 / 为 / 显示"）。其中，**开篇核心小卡、模块1「现在有多热」、模块3「谁在动、谁没动」里的所有结构化板块 / 个股行情、个股估值、成交、资金、财务数字**，必须来自 `seed_finance_search`；其它固定结构化展示里的证券市场行情 / 个股估值 / 成交 / 资金等市场数字，以及上市公司个股收入、利润、毛利率等业务 / 财务数据，也必须来自 `seed_finance_search`（同花顺数据库）或经其核验。行业层面的经营、旅游消费、订单需求、第三方统计等非个股事实，不要求一律来自同花顺，可使用一级或二级来源并显名限定。模块2 催化优先使用一级，也允许符合要求的二级来源（须显名归因与限定措辞），禁止三级。

### 第二级 · 可引用，但须标注来源、措辞需限定

- 可核验的机构媒体 / 新闻网站的**原创或转载报道**（采编主体清晰，但非一手发布）——含财经媒体与综合新闻媒体，如财联社、证券时报、上海证券报、中国证券报、第一财经、界面、华尔街见闻、财新、21 世纪经济报道、光明网、新京报、中国经济网等；
- 行业协会与第三方机构数据（如上海有色 SMM、中汽协、乘联会、SNE Research、克而瑞等）；
- 券商**一致预期 / 估算**（是机构测算，而非披露事实）；
- 平台机构号 / 官方号（如雪球、今日头条、微博、公众号等平台上的机构媒体号、上市公司号、行业协会号、第三方机构号、券商号；前提是主体清晰，非个人 / 自媒体）；
- 一般新闻对一手信息的**转载报道**。

上述二级来源名称是**非封闭示例名单，不是封闭名单**；其它可核验、非三级的平台 / 机构，只要有清晰来源名、原文链接，并能合理标注 `source.tier:2`，也可以作为模块2 催化来源。

**平台页判定原则**：看署名主体，不看域名。机构主体可直接标二级，`source_name` 填机构名；个人 / 自媒体 / 普通用户 / 无法确认机构主体标三级，不得进模块2。比如雪球 / 今日头条 / 东方财富页面，作者是券商、机构媒体、上市公司等机构号时，可作为二级直接使用；拿到真正官方 / 公告 / 原文链接时，才另按一级处理。

二级数据引用时必须**显名归因并限定措辞**——"据 XX 媒体报道""据券商研报估算""据中汽协数据"，用"据 / 或 / 可能"，**不得写成一手披露的确定事实**。**证券市场行情 / 估值 / 成交 / 资金等市场数字，以及上市公司个股营收 / 利润 / 毛利率等业务与财务数字，即便出现在二级来源里，也不能直接进入开篇核心小卡、模块1、模块3或其它结构化展示，只能作为线索去 `seed_finance_search` 核验后再用数据库的数据**；行业层面的经营、旅游消费、订单需求、第三方统计等非个股数据可以引用二级来源，但必须显名和限定。模块2 的催化叙事中，行业数字、市场情况、业务进展、资本市场流向可来自一级 / 二级原文；若同一数字要进入结构化行情展示或用于模块1 / 模块3 / key_chips，则必须回到 `seed_finance_search` 登记为一级。

### 第三级 · 不撑展示数值；**禁止**作模块2「为什么涨/跌」的新闻来源

- 个人号、自媒体号、普通用户发帖 / 评论、UGC 内容（头条号、微博个人号、百家号个人作者、雪球个人账号、股吧普通用户、各类博主 / 个人专栏，**尤其个人作者发表的文章**）；
- 行情摘要、研报摘要、无署名转载、无法确认机构主体的交易社区 / 平台 / 聚合 / 分发页；
- 移动分发页 / 跳转页 / 聚合转载页中**署名为个人、自媒体、号主、普通用户或无法确认机构主体**的内容（如头条号、微博详情页、百家号个人作者、新浪自媒体 `/wm/` 等）；
- 市场传闻、社交讨论、无署名来源。

三级信息**不得作为证券市场行情 / 估值 / 成交 / 资金等市场数字的依据**，且**禁止作模块2 催化的信息来源**。聚合 / 分发平台可以用来找线索；最终入选时，只保留一级来源或主体清晰的二级机构来源。平台上的机构号按二级，个人 / 自媒体 / 普通用户 / 无法确认机构主体按三级。

### 一句话记忆

开篇小卡 / 模块1 / 模块3及其它结构化展示里的证券市场数字 ＝ `seed_finance_search`；模块2催化 ＝ 一级或二级 + 可打开链接；平台页先看署名主体，机构主体按二级，个人 / 自媒体 / 普通用户 / 无法确认机构主体按三级。

### 分级例子与措辞速查

| 等级 | 能否作证券市场数字 / 个股财务数据 | 能否作非个股行业事实 | 能否作模块2 催化 | 正文措辞 | facts `usage_type` |
|---|---|---|---|---|---|
| 一级 | 可；结构化市场数字与上市公司个股财务数字必须用数据库 / 经数据库核验 | 可，官方 / 公告 / 权威数据可直接陈述 | 可，优先 | "披露 / 为 / 显示 / 公布" | `hard_fact` / `official_statement` |
| 二级 | 不直接展示；只作线索回到 `seed_finance_search` 核验 | 可，须显名归因和限定 | 可，显名 + 原文链接 | "据 XX 报道 / 估算 / 数据" | `media_view` / `broker_estimate` / `industry_data` |
| 三级 | 不可 | 原则上不可作正文依据 | 禁止作模块2最终来源 | 仅内部留痕，不进正式事件 | `rumor_lead` |

一级例子：

- `seed_finance_search`：板块 / 个股收盘价、涨跌幅、成交额、换手、量比、资金流、个股 / 代表股 PE(TTM) / PB / PS、涨停 / 上涨家数、上市公司个股营收 / 利润 / 毛利率等。
- 官方与一手原文：政府 / 监管 / 交易所 / 央行 / 统计局 / 公司公告 / 年报 / 季报 / 投资者关系活动记录 / 新华社等官方通讯社 / 券商研报原文。

二级例子：

- 机构媒体 / 新闻网站：财联社、证券时报、上海证券报、中国证券报、第一财经、界面、华尔街见闻、财新、21 世纪经济报道、光明网、新京报、中国经济网等。
- 行业协会 / 第三方机构：上海有色 SMM、中汽协、乘联会、SNE Research、克而瑞、QuestMobile 等。
- 券商一致预期 / 估算、平台机构号 / 官方号、一般新闻转载。

三级例子：

- 个人号 / 自媒体号 / 普通用户发帖：头条号、微博个人号、百家号个人作者、大鱼号、雪球个人账号、股吧普通用户、各类博主 / 个人专栏。
- 行情摘要、研报摘要、市场传闻、社交讨论、无署名来源。
- 无法确认机构主体的平台 / 聚合 / 分发页；平台上的机构号 / 官方号不因平台域名自动判三级。

## 第二部分 · 七条硬规则

### 规则 1 · 锁定唯一目标概念板块，板块指标只用板块自身数据

1. **先广搜主题归因，再锁概念板块**：拿到用户的板块 / 概念 / 题材后，先按 `references/data_contract.md` 做宽口径主题搜索，形成 3-5 条近期表现归因假设；`general_search` 每个关键词 / 查询最多读取 5 条搜索结果，超过 5 条不继续展开；该限制不适用于 `seed_finance_search` 的行情 / 成分股 / 10 股取数。这些只作候选判断，不是正文结论。
2. **再搜概念板块、择全确认**：在已有主题判断基础上，找出**对应的、带代码的概念板块**（同一主题常有多个近义板块，如"低空经济 / eVTOL / 通用航空"），并分别搜索行情与资料，**以"目标最贴合且 5 个板块核心指标可完整取得"的那个作为目标概念板块**（名称 + 代码），全篇只围绕这一个板块展开，不在多个板块间游移。
3. **板块核心指标必须取目标板块自身**：收盘点位、前收、当日成交额、近7日收盘基准与 7 个交易日成交额必须来自目标概念板块自身口径，并登记进 `sector_checks`；当日涨跌幅、近7个交易日涨跌幅、近7个交易日日均成交额由 `derive_facts.py` 基于这些原始分量计算。不能用 10 只代表股合计 / 均值替代，也不能把替代值写成板块值。
4. **按“相关性入池 + 市值排序”选 10 只代表股**：代表股必须来自该板块成分股。先剔除主业弱相关、仅概念沾边、无法说明代表性的公司；剩余股票仍按市值从大到小取前 10 只。不得为了追求主题纯度而选入明显小市值、低流动性、非代表性的边缘公司。选出的 10 只只作为模块3代表股表格、代表股表现、股票 PE(TTM) 与分化分析样本，每只写一句"为什么选它"的理由。

若目标概念板块自身的 5 个核心指标拿不全，回到 `seed_finance_search` 补取，或改选更贴合且数据完整的目标概念板块；仍无法取得时，不输出这些板块指标。

### 规则 2 · 开篇 / 核心结论显式标注目标板块 + 10 只代表股

- **对话框 md**：正文**第一段**必须明确写出"目标概念板块：XXX（代码）"；该段结束后另起一行，用**灰色小字**注释"本次选取的 10 只代表股：股票1、股票2…"（共 10 只）。
- **飞书文档**：核心结论处显式标注目标概念板块；核心结论文字之后另起一行小字，注释这 10 只代表股名称。（`index_caliber` → 核心结论处的目标概念板块说明；`selected_stocks` → 核心结论下的小字注释。）

### 规则 3 · 代表股表格只展示按“相关性入池 + 市值排序”选出的这 10 只，且给出选取理由

- 模块3 的代表股表格**只展示目标概念板块内按“相关性入池 + 市值排序”选出的 10 只**（不多不少）；这 10 只在第一阶段 `facts.json.stock_checks` 中确定，文档生成阶段由脚本自动映射到 payload 的 `stocks` 数组。
- **每只配一句"选取理由"**（第一阶段写进 `facts.json.stock_checks[].select_reason`，文档生成阶段自动映射到 `stocks[].select_reason`，回退 `note`），可在飞书表格或补充说明中展示，用一句话说明为什么选它（如"铜系龙头，矿端紧缺逻辑最硬"），不带涨跌幅 / 成交额等数据数字。

### 规则 4 · 每个展示数值都要分级、溯源、登记

- **分级**（第一部分的执行）：除模块2 催化外，证券市场行情 / 估值 / 成交 / 资金等市场数字（价格、涨跌幅、成交额 / 换手 / 量比、估值、区间高低点、涨停家数、代表股涨跌 / 成交等）以及上市公司个股收入 / 利润 / 毛利率等业务与财务数据，一律是**一级数据**——只取自 `seed_finance_search`，或经其核验。出现在新闻 / 二手源里的此类数字只能作线索去核验，**未经数据库核验的二手市场数字 / 个股财务数字一律不引用、对应的二手源也不引用**。行业层面的经营、旅游、订单 / 需求、第三方统计等非个股事实可用一级或二级来源，须显名归因与限定措辞。
- **溯源**：第一阶段 facts.json / 取数笔记要登记来源；文档 payload 里 `dimensions` / `key_chips` / `stocks` / `catalysts` 的每一项都要带 `source`（至少 `lane`；行情项建议带 `as_of`；催化必须带 `level` 与 `url`）。文档 payload 校验门会逐项核对，缺 `source`、行情项标错来源、催化缺级别 / 原文，都会拒绝写入飞书文档。
- **登记**：每个展示数字都在 facts.json 里有一条对应登记（值 + tier + source_name；可复算项带分量），并由复算门真的算一遍对账。板块与 10 股行情还必须在 `sector_checks.evidence` / `stock_checks[].evidence` 粘贴 `seed_finance_search` / 同花顺返回的原始数据行摘要；lint 不联网验真，但没有取数留痕的自洽数字不能通过。

### 规则 5 · 模块2「为什么涨 / 跌」必须显名归因、可打开原文，且来源只许一级 / 二级

- **每条催化都要显名归因并可核验**：模块2 的每条催化必须记录来源名与可打开原文链接（第一阶段 facts / 取数笔记登记，文档 payload 对应 `source_name` 与 `url`），二者缺一不可。这里约束的是**数据真实性与可追溯性**；飞书文档中如何展示来源标签，归 `references/payload_fields.md` 与 `references/feishu-doc-style.md` 管。
- **时效性是裁定条件**：模块2 催化尽可能选行情截止日前 14 个自然日内的事件，优先最近 7 个自然日 / 最近 5 个交易日内的新闻。超过 14 天的旧催化只有在政策 / 监管 / 官方文件 / 公司公告 / 财报或行业数据仍在兑现，或本周被行情重新交易时才可保留，并在第一阶段 facts / 取数笔记中登记 `freshness_reason`，文档 payload 复用该理由；否则即便来源为一级 / 二级，也只作背景或内部线索，不进模块2。
- **来源只能选取数据来源为一级和二级的数据、禁止三级**——一级：政府 / 人大 / 监管 / 交易所 / 央行 / 统计局 / 官方通讯社 / 公司公告 / 政策原文 / 券商研报原文（巨潮、沪深交易所、证监会、央行、统计局、新华社等）；二级：可核验的机构媒体 / 新闻网站报道（含财经媒体与综合新闻媒体，如财联社、证券时报、第一财经、光明网、新京报、中国经济网等）、行业第三方（如上海有色 SMM、中汽协、乘联会等），以及身份清晰的平台机构号 / 官方号。**二级名单是非封闭示例，不是封闭名单**；其它可核验、非三级的平台 / 机构也可标 `source.tier:2` 使用。**禁止明确三级**（个人号 / 自媒体号 / 普通用户发帖 / 评论 / UGC / 行情摘要 / 研报摘要 / 无署名转载，或无法确认机构主体的平台 / 聚合 / 分发页）：搜索时可先用聚合 / 分发平台找线索，再为已锁定归因寻找一级 / 二级来源；若聚合承载页或平台账号的署名 / 作者 / 账号主体 / 认证信息 / 采编主体明确是机构媒体、新闻网站、上市公司、行业协会、第三方机构或券商，可按该机构作为来源保留并补 `source.tier:2`，不要求继续寻找原站；若署名为个人、自媒体、号主、普通用户或无法确认机构主体，就不写这条催化。
- 可显式标注 `source.tier`（1 / 2）以便校验；不标时按来源名与 url 识别（命中个人作者 / 自媒体黑名单即判三级、纵使误标更高级也以黑名单为准；聚合承载页仅在署名 / 账号主体 / 认证信息 / 采编主体为机构时按二级放行但给警告，标一级则报错）。
- 一句话：模块2 **只用一级 / 二级、禁止三级**，并**强制显名归因 + 强制可打开原文核验**；平台 / 聚合承载页看页面署名 / 作者 / 账号主体 / 认证信息 / 采编主体，机构主体直接按二级入选，个人 / 自媒体 / 普通用户 / 无法确认机构主体不得作催化来源。

### 规则 6 · 模块2 的事实总结只依据来源原文

- 模块2 每条催化的**事实总结（`fact`）只依据该来源原文概括**，**禁止掺入其它数据或分析**（哪怕来自别处、哪怕是对的，也不行）。
- "为什么重要 / 后续验证"是基于该事实的推断，要明确标注为分析、与事实分层（见 `references/chat_contract.md`），同样不得引入来源原文之外的新"事实"。

### 规则 7 · 行情数据口径锁定在「唯一目标板块 + 10 只代表股」之内

锁定目标板块与 10 只代表股后，**全篇涉及"板块涨跌 / 个股涨跌"的行情数据，只能取自这个目标板块本身、或这 10 只代表股**。

- **禁止**在任何位置写入**其它概念板块**的行情数字，也**禁止**写入**这 10 只以外个股**的涨跌 / 资金 / 成交。需要佐证时用目标板块或这 10 只的数据支撑。
- **允许的对照仅两类**：①目标板块自身的相对位置 / 排名（如"主力净流入居 31 个申万一级行业首位"）；②模块2 的宏观 / 政策 / 地缘 / 产业等**非板块行情类**催化（非农、利率、油价、政策、产业新闻等，一级 / 二级均可）——这些是驱动事件，不是"别的板块 / 别的个股的行情数字"。
- 结构层面已由代码强制（`stocks` 必须是选定的 10 只、飞书表格只展示这 10 只）；本规则补齐的是**文字层面的口径一致性**：`dimensions` / `key_chips` / `headline` / `summary` / `catalysts` 里的板块与个股行情数字，都必须落在上述口径内。
