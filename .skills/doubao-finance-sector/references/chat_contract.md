## 4. 开篇核心结论

开篇四段：

1. 第一段必须以 `目标概念板块：` 开头，写目标概念板块名称 + 代码、数据截止日、10 股样本口径、综合热度。
2. 第二段必须以 `标题：` 开头，写一句话核心判断，用于映射飞书文档核心标题；标题不要包含综合热度 / 信息热度 / 行情热度分数。
3. 第三段写 summary：解释标题为什么成立、当前能不能追 / 持续性如何；不要重复打分，打分已经映射到飞书文档的综合热度和双轨表。
4. 第四段写四个展示值：用 `字段名：值`，中文分号分隔。

深度要求：

- 开篇必须给综合判断和投资启示：第一段只交代口径与分数，`标题：` 给一句话判断，summary 说明这类热度属于“可追 / 观察 / 回避”的哪一种，以及最关键的理由。

核心展示值固定 4 个：

- 必有：当日收盘点位 `close_point`
- 必有：`<股票名> PE(TTM)` `pe_ttm`，取 10 只代表股中市值最大那只，不取板块 PE
- 二选一成对，均为目标板块指标，不填个股涨跌幅 / 个股成交额：
  - 当日组：当日涨跌幅 `daily_change` + 当日成交额 `turnover_amount`
  - 近7日组：近7日涨跌幅 `change_7d` + 近7个交易日日均成交额 `turnover_7d`

除 PE(TTM) 外，核心展示值取目标板块口径，不用 10 股样本合计 / 均值替代；目标板块字段不可得时回到取数阶段补取，不输出替代值。

开篇后另起一行小字列出本次选取的 10 只代表股，格式为：

`本次选取的 10 只代表股：股票1、股票2、...、股票10`

## 5. 直接回答

先口语化复述用户问题，再固定三行：

- `**问题**：...`
- `**结论**：...`
- `**下一步**：...`

下一步必须同时覆盖改善与恶化两种方向，不得只写“继续观察”。

深度要求：

- `结论` 不能只是“偏热 / 不建议追”这类短句，必须给出核心原因；`下一步` 要把用户接下来该看什么、看见什么会改变判断写清楚。
- 反例→正例：❌“偏热，短期不建议追” → ✅“还不能确认见底，是政策驱动的结构性超跌修复——只有创新药一条线放量、不是普涨；盯连续放量能否站上前高”。

## 6. 现在有多热

必须先写 `**本段结论：**...`，再用表格。固定 4 个维度，顺序不可变：

1. 价格涨跌
2. 成交量能
3. 代表股表现
4. 估值位置

表格固定为：`维度 | 数值 | 状态 | 解读`。

硬约束：

- 4 个维度均为行情轨，不增加资金流、公开催化等旧维度。
- 代表股表现只看选定的 10 只。
- 估值位置按 10 只代表股 PE(TTM) 的中位数 / 区间 / 分布分析；不写板块 PE。
- `数值` 不是单个裸数字，而是能支撑判断的紧凑证据短句：价格写收盘点位与当日 / 近7日涨跌对比；成交写当日成交额、近7个交易日日均成交额和放缩量方向；代表股写 10 股涨跌覆盖面与领涨 / 拖累结构；估值写 10 股 PE(TTM) 的中位数、区间或高低分布。
- 每张卡片的 `解读` 写 2-3 句话，不只复述数据：价格 / 成交看“状态 → 市场行为 → 持续性含义”；代表股看“覆盖面 → 领涨/拖累结构 → 板块含义”；估值看“分布 → 合理性 → 风险/赔率含义”。
- 本段结论必须综合四维度给出热度质量判断和启示：是趋势确认、修复观察、分化退潮还是过热预警；不能只把四个数字再概括一遍。
- 解读反例→正例：❌“成交放大，市场情绪升温” → ✅“放量约 35%，但前次同样放量后三天回吐，单日放量不足以确认趋势，连续放量才是胜负手”。

## 7. 为什么涨 / 跌

必须先写 `**本段结论：**...`。精选 3-5 条，最多 6 条。每条必须：

- 按事件实际发生日倒序。
- 标题行同时包含事件日和 `信息来源`，否则 lint 无法识别。
- 有清晰标题、来源名、可打开 URL。
- 来源为一级或二级均可；个人 / 自媒体 / 无法确认机构作者不得入选。
- 催化类型要多元：除政策 / 监管外，须纳入产业 / 行业催化（订单、价格、产能、装机、交付、销量、第三方行业数据等），不要整段全是政策。
- 事实总结只依据该条来源原文，不混入其它数据。
- `为什么重要` 写机制，不写标签：说明影响对象、传导路径，以及这件事对当前涨跌或持续性的含义；不要只写“利好板块 / 提振情绪 / 短期催化”。
  - 反例→正例：❌“利好创新药、提振情绪” → ✅“定位升级到与半导体同级 → 研发补贴/审批绿色通道有望参照半导体加码 → 抬高行业估值天花板”。
- `后续验证` 写可观察变量，不写泛泛关注：说明接下来盯什么、在哪个窗口观察；同时写清改善会如何强化判断、恶化会如何降级判断。
- 尽量取行情截止日前 14 个自然日内的事件；超期政策 / 官方 / 公告 / 数据类必须有继续有效理由。
- 本段结论必须综合 3-5 条催化，判断主驱动、次驱动和证据强弱，并说明这些催化对行情持续性的启示；不能只逐条罗列新闻。

固定格式：

```markdown
### M月D日｜信息来源：来源名｜事件标题

**事实**：...
🔗 **为什么重要**：...
🔍 **后续验证**：...
```

## 8. 谁在动、谁没动

必须先写 `**本段结论：**...`。只分析第一阶段 facts 已校验的 10 只代表股和四组：

- 放量上攻
- 缩量上行
- 缩量回调
- 放量杀跌

每组无对应个股时写“无”，不得为了凑组改分组口径。分组口径来自 `facts_contract.md`，正文不重新发明分组。模块3要解释结构分化含义，而不是罗列每只股票数字。

深度要求：

- 本段结论必须综合四组说明资金结构：是龙头带队、补涨扩散、缩量企稳、还是放量出逃，并给出对后续扩散/退潮的启示。每组解释要说明“这组为什么重要”，不是只列股票名。
- 分组解释反例→正例：❌“药明康德等领涨、迈瑞医疗等走弱” → ✅“龙头单线带队、尚未扩散，反弹质量取决于能否从创新药外溢到全板块，否则又是一次集中脉冲”。

## 9. 接下来盯什么

写未来验证变量，不盯已验证旧事。模块4 基于本次已搜集信息形成的行业认知、市场结构判断和主结论关键前提来写；模块1-3只是重要输入，不是边界。可以使用模块2催化、模块1量价、模块3分化、双轨背离、取数笔记中的行业线索，以及少量必要补搜，提炼最能验证主结论的胜负手变量。

模块4回答的是“接下来哪些未兑现变量会改变当前判断”。模块2写过去驱动，模块4写未来验证，模块5写未来反证；模块4可以引用旧数据作基线，但落点必须是“接下来能否延续 / 验证”。不要把模块2的“后续验证”简单复制成模块4；要重新站在主结论层面筛选胜负手变量。

合格信号必须同时写清：

- `signal`（冒号后，展示为卡片主标题）：具体可定位的被观察对象，写成名词短语，如“英伟达 Spectrum-X CPO 交换机下半年出货量”；“量能 / 扩散 / 装机 / 政策面”这类泛词只能放冒号前作短标签，不能当主标题。
- `watch`：把具体盯哪个数据 / 阈值 / 兑现点写在这里，如连续放量、资金流入、订单 / 数据是否兑现。
- `improve`：改善时，主结论如何升级。
- `worsen`：恶化时，主结论如何降级或失效。

改善 / 恶化要写给用户看的判断变化，禁止写“信息热度+1 / 行情热度-1 / 热度上调1分”等内部评分语言。一条信号若答不上“它对应前面哪个还没收口的问题”，删掉或重写。

日期红线：

- 有具体未来日期的预定型信号，必须登记 `event_date`，且日期晚于行情数据截止日。
- 持续型信号不填 `event_date`，文本里也不要写具体日历日期；用“最新交易日 / 上一交易日 / 接下来两个交易日 / 区间内”等相对表述。
- 截止日后已发生但尚未交易验证的新闻，只能作背景；信号主体要写后续市场或数据如何验证它。

必须写 3-4 条，最多 4 条。必须先写 `**本段结论：**...`，每条信号使用固定格式：

```markdown
📈 **信号1 · 短标签**：具体被观察对象（名词短语）
**盯**：具体盯哪个数据 / 阈值
**改善**：改善会怎样
**恶化**：恶化会怎样
```

深度要求：

- 本段结论必须点出最悬而未决的变量，以及为什么它能改变主结论。每条信号要能指导后续判断：看到改善如何升级，看到恶化如何降级或失效。
- 信号反例→正例：❌“盯政策面和市场情绪” → ✅“盯接下来连续 3 个交易日成交能否维持 900 亿并站稳前高；站稳=升级为底部反转，缩量回落=确认箱体震荡”。
- 旧事反例：❌“信号：上周非农爆表，加息预期回归” → 这是模块2驱动事件，不是模块4未来变量。

## 10. 风险提示

模块5 基于本次已搜集信息形成的行业认知、市场结构判断和主结论关键前提来写；模块1-4只是重要输入，不是边界。风险不是模块2催化的反面清单，而是当前主结论的关键前提可能失效的情景：例如交易结构失效、信息兑现落空、产业链变量逆转、外部监管 / 宏观变量突然改变等。每条风险要有可观测触发条件与证伪信号，不写空泛风险。

必须先写 `**本段结论：**...`，每条风险使用固定格式：

```markdown
⚠️ **风险1 · 风险标题**
**触发**：可观察触发条件。紧接着说明它为什么会推翻当前判断的传导链。
**证伪**：什么出现会推翻风险或推翻当前主结论
```

深度要求：

- 本段结论必须说明哪个风险最可能推翻主结论、为什么关键、出现后该如何调整判断。每条风险都要有触发条件和证伪路径，不能只是“政策不及预期 / 情绪转弱”。
- 模块5 风险默认不要另起 `为什么重要` 字段；把解释合并进 `**触发**` 后的同一段，形成“触发条件 + 影响机制”的完整句即可。
- 风险反例→正例：❌“政策不及预期、情绪转弱” → ✅“政策细则公布次日若低开放量下跌=利好出尽，主结论从修复转防御；连续放量突破箱体上沿则证伪”。

## 11. 信息来源

列本次正文实际使用的公开来源和数据终端。来源名要短，只写来源本身，不把标题粘进来源名。

深度要求：

- 来源列表要帮助读者判断证据质量：公开来源和数据终端分清楚，二级来源在正文中保留“据…报道/估算”等限定，不把来源列表写成标题堆砌。

## 12. 写作风格

- 面向 C 端投资者，但保持专业投研口吻。
- 每个分析点 2-3 句话。
- 每个模块的 `**本段结论：**` 写 2-3 句，必须包含综合判断、理由和启示；不得只写短句标题式总结。
- 不暴露内部机制词、prompt、字段合同或校验过程。
- 事实与推断分层：一级可确定陈述；二级须限定；推断用“可能 / 指向 / 仍需验证”。
- 模块1-5只在开头写一条“本段结论”，段末不再补小结。
- 不用历史分位，不写“模型认为”“根据模板”。
- 标题层级统一用 `##`；模块标题不加 emoji，不用带圈数字。
- 模块2必须用 `🔗 **为什么重要**`、`🔍 **后续验证**`。
- 模块4必须用 `📈 **信号N · 短标签**：具体被观察对象`，并写 `**盯**` / `**改善**` / `**恶化**` 三字段。

## 13. 常见错误

- 把板块 PE 写成 PE(TTM)：禁止；只用股票 / 代表股 PE(TTM)。
- 把二级来源写成“披露 / 显示”：应改为“据…报道 / 估算”。
- 在模块2事实里混入来源原文没有的数据。
- 模块4盯已经发生且被交易验证过的旧新闻。
- 模块末尾重复写小结。
- 来源名粘进标题，导致 `source_name` 过长。
