# 数据合同（默认读取）

本文件是第一阶段取数与分级的默认合同。长解释、示例名单、措辞速查与边界案例见 `data_rules.md`；除非报错或分级不确定，不要默认读取长参考。

## 1. 两条来源 lane

| lane | 用途 | 能否进结构化展示 |
|---|---|---|
| `seed_finance_search` | 同花顺数据库里的证券市场行情、成交、资金、个股 / 代表股估值、上市公司个股财务数字 | 是，固定模块的结构化市场数字必须用它 |
| `general_search` | 公开原文、新闻、公告、政策、行业数据、模块2催化、模块4/5背景 | 只用于公开事实与催化；新闻里的证券市场数字只能作线索，展示前必须回到 `seed_finance_search` |

用 `seed_finance_search` 取数时优先命中：个股用标题带“最新实时股票信息与股票行情走势”的结果；板块用“指数每日全面行情”。

硬红线：

- 原始行情 / 财务数字必须由 `seed_finance_search` 实际搜索取得，**禁止凭估计 / 估算、记忆或经验补值，也禁止用相近数推断**；取不到就按缺口 / `data_mode` 处理，不许填一个“差不多”的数。
- 开篇核心小卡、模块1、模块3及其它固定结构化展示里的板块行情、个股行情、个股 / 代表股估值、成交、资金、上市公司个股财务数字，只用 `seed_finance_search`。
- 模块2 的行业数字、市场情况、业务进展、资本市场流向可来自一级 / 二级公开来源，但如果进入结构化行情展示，必须回到 `seed_finance_search`。
- 禁止采集、回填或展示板块 PE / 板块市盈率；PE(TTM) 只填股票 / 代表股。

## 2. 可信度分级

| 等级 | 判定 | 可用边界 | 正文口吻 |
|---|---|---|---|
| 一级 | 原始 / 权威来源：`seed_finance_search`、官方、监管、交易所、公司公告、官媒、一手研报原文 | 可作展示事实；结构化市场数字优先用 `seed_finance_search` | “显示 / 披露 / 为” |
| 二级 | 机构二手来源：机构媒体、新闻网站、行业第三方、券商估算；或聚合平台上的明确机构号 / 公司号 / 协会号 / 券商号 | 可作模块2催化；行业非个股事实可用 | “据…报道 / 据…估算 / 可能” |
| 三级 | 非机构或无法确认主体：个人号、自媒体号、普通用户、UGC、行情摘要、传闻、无署名转载 | 只作线索；不得作展示值或模块2最终催化 | 不进正文事实 |

聚合 / 分发平台不按域名定级，只看署名主体：机构主体按二级，`source_name` 填机构名；个人 / 自媒体 / 普通用户 / 无法确认机构主体按三级。例如雪球 / 今日头条 / 东方财富页面，若作者是券商、机构媒体、上市公司等机构号，可标二级直接用。

## 3. 搜索与记录流程

`general_search` 搜索预算：每个关键词 / 查询最多读取 5 条搜索结果；如 5 条内无法形成判断，换关键词或缩窄问题，不继续展开同一关键词的长结果列表。此限制只约束公开搜索，不约束 `seed_finance_search` 的行情、成分股、10 股与近7个交易日取数。

全局时间锚：`T` / `T0` 固定为“最近一个已经完成收盘的交易日”。若在交易日盘中生成，不能把当日未收盘行情作为 `T`；应使用上一已收盘交易日作为 `T`，并在 `meta.timestamp` / 取数笔记中写成“数据截至 YYYY-MM-DD 收盘”。

取行情前必须先确定 `T0`；未确定 `T0` 前不得搜索或登记行情数字。下文所有“当日”均指 `T0` 这个数据截止日的已收盘行情，不是自然日今天、当前时刻、实时价或盘中价。

近7日相关数据第一轮取数就要按 `T-7` 至 `T` 的 8 个已收盘交易日一次性拿齐，不要先搜“最近7个交易日”再回头补 `T-7`。同一批日线里，`change_7d` 用 `T-7` 收盘与 `T` 收盘复算；`turnover_7d` 只截取 `T-6` 至 `T` 的 7 个成交额求均值。

1. 先用 `general_search` 宽口径形成 3-5 条近期归因假设，只作线索。
2. 用 `seed_finance_search` 直接检索目标概念板块、10 只代表股、板块与个股原始行情字段、股票 PE(TTM)；不得用新闻摘要、记忆或自造序列填写行情字段。取数时优先命中：个股用标题带“最新实时股票信息与股票行情走势”的结果；板块用“指数每日全面行情”。
3. 用行情验证归因是否解释板块和 10 股的强弱、量能和子方向分化；不吻合时再定向补搜。
4. 只为最终进入模块2的 3-5 条催化找一级 / 二级 URL；未入选线索不做完整溯源。
5. 模块4/5 基于本次已搜集信息形成的行业认知、市场结构判断和主结论关键前提来写；模块1-3只是重要输入，不是边界。可使用取数笔记中的行业线索，并在缺未来变量或证伪触发条件时少量定向补搜。

## 4. 必采数据清单

第一阶段至少采集并登记：

- 目标概念板块身份：名称 + 代码 / 口径 + 数据截止日。
- 目标板块 `T0` 已收盘原始字段：`close_point`、`prev_close`、`turnover_amount`。
- 目标板块近7个交易日原始字段：`d7_close_base`（T-7 交易日收盘点位）、长度为 7 的 `d7_turnovers`（T-6 至 T 的 7 个交易日成交额）；`d7_close_t` 由脚本用 `close_point` 补齐。
- 板块与个股每行行情取数证据：在 `sector_checks.evidence`、`stock_checks[].evidence` 粘贴 `seed_finance_search` / 同花顺返回的原始数据行摘要；可在取数笔记或 evidence 里保留命中的结果标题 / 栏目，但不作为 lint 硬校验。没有证据的自洽数字不能通过。
- 10 只代表股身份：取自目标板块成分，按“相关性入池 + 市值排序”选出；逐只登记股票名称 / 代码、成分股归属、市值排序依据、`role` 与 `select_reason`。
- 10 股 `T0` 已收盘字段：逐只登记 `change`、`turnover`、股票 `pe_ttm`；`pe_ttm` 为必填，不得写板块 PE。
- 10 股近7个交易日原始字段：逐只登记 `d7_close_base`（T-7 交易日收盘价）、`d7_close_t`（T 日收盘价）、长度为 7 的 `d7_turnovers`（T-6 至 T 的 7 个交易日成交额）。
- 派生字段：不要手填 `daily_change`、`change_7d`、`turnover_7d` 或 `divergence_groups`；运行 `python3 scripts/derive_facts.py work/<板块>_facts.json` 统一计算。
- 核心展示数字：收盘点位、`<股票名> PE(TTM)`、以及当日组或近7日组成对两项；除 PE(TTM) 外都必须是目标板块自身口径，不得用 10 股样本合计 / 均值替代。
- 模块2催化：精选 3-5 条、最多 6 条，每条有事件日、来源名、tier/level、可打开 URL、事实摘要。

## 5. 取数笔记建议

建议把搜索过程沉淀到 `work/<板块>_取数笔记.md`，不要都塞进对话上下文：

- `general_search/core_attribution_hypotheses`：归因假设、线索来源、验证状态。
- `general_search/public_evidence`：最终入选催化的来源、tier、URL、裁定。
- `seed_finance_search/sector_snapshot`：板块收盘、涨跌、成交、资金 / 宽度等；禁止板块 PE。
- `seed_finance_search/representative_valuation`：最大市值代表股 PE(TTM)、10 股 PE(TTM) 中位数 / 区间。
- `seed_finance_search/representative_stocks_daily`：10 股 `T0` 已收盘涨跌、成交额、PE(TTM)、角色和选取理由。
- `seed_finance_search/representative_stocks_7d_history`：10 股逐只一次性取 `T-7` 至 `T` 的 8 个已收盘交易日日线；从中登记 `T-7` 收盘价、`T` 收盘价，以及 `T-6` 至 `T` 的 7 个交易日成交额序列。
