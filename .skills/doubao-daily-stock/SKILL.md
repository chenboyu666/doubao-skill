# doubao-daily-stock

## 适用场景

本 skill 用于单一上市股票或二级市场公司的当日/近期个股日报，重点分析行情表现、涨跌原因、资金变化、新闻公告、板块联动、技术面、预期与风险。

适用于用户询问“某股今天为什么涨跌”“做个日报”“近期表现如何”“资金面和消息面怎么样”“生成飞书文档版完整报告”等场景。核心判断标准是：单一公司 + 短期动态 / 行情复盘。

不用于长期商业模式/护城河、财报业绩深度分析、行业研究、板块热度、多股主题、一级市场或大盘事件解读；遇到这些问题应转向相应 skill 或先澄清。

默认先获取数据并输出结构完整、观点深入的对话版个股日报；只有用户明确同意写入飞书文档时，才进入飞书文档编辑流程。

## 数据准确性最高优先级

这是一份严谨金融分析报告。数据准确性高于版式、速度和观点表达；只要一个关键数据错误，整份报告就失去权威性和意义。

执行本 skill 时必须先完成“数据验收”，再进入分析写作和飞书文档版生成：

- 所有关键行情、财务、资金、板块、技术指标、新闻、公告、研报/一致预期数据，必须来自 `seed_finance_search` 明确返回的金融数据，或 `general_search` 检索到的官方/权威机构来源。当前版本采用“权威单源 + 清晰口径”验收；只要来源权威、证券代码和市场一致、时间口径和单位清楚，即可使用。每一个进入报告的数值都必须记录来源链接或工具来源标识；不是模型自己计算的数值，正文和飞书文档版中必须用来源编号回链到“信息来源”。
- 摘要指标 8 项和 A 股资金流向指标是强制专用字段，只能来自 `seed_finance_search（同花顺数据库）` 或基于其返回值自行计算。摘要指标 8 项包括当前价格/收盘价、涨跌幅、日内区间、总市值、市盈率 TTM、成交额、换手率、量比；其中量比只在收盘口径下自行计算，公式为 `当日成交量 / 过去 5 个交易日平均成交量`，输入数据必须来自 `seed_finance_search（同花顺数据库）`；若为盘中口径，量比统一写 `暂无`，不自行估算。A 股资金流包括主力/超大单/大单及陆港通标的北向资金，不得用 `general_search` 或其他来源兜底。港股/美股资金口径为资金净流入/净流出；先查 `seed_finance_search`，若未返回，可用 `general_search` 检索官方或权威机构来源，但数据日期必须是报告当日或最近一个有效交易日；更早数据视为无效并写 `暂无`。
- 如果数据存在冲突、口径不明、日期不对、数量不足或无法核验，宁可留空；表格和摘要指标统一写短占位 `暂无`，不得写“暂未获取可信信息”这类长句，也不得猜测、补造或沿用疑似错误数据。
- 必须先判断用户发起请求的时间点，并据此确定 `report_request_time`、`data_as_of` 和最近有效交易日。非交易日请求时，应向前推到最近一个真实交易日；若该日停牌或休市，继续向前推。
- 取数阶段必须先形成内部数据核验记录，至少包含：证券身份、报告时点、数据截至时点、每一个行情/财务/新闻/技术指标字段的数值、单位、来源名称、原始外部来源链接或 `seed_finance_search（同花顺数据库）` 工具来源标识、来源权威性判断、缺失项、自己计算项及公式。该记录只用于工作流内部核验，不要求保存为本地 `.txt`，也绝不能作为报告末尾“信息来源”。后续分析和飞书文档版只能使用已核验数据；若必须新增材料，也必须先补充核验后再使用。
- 当前版本不获取 1 分钟级行情，不获取 30 个交易日日线，不获取 90 个交易日行情。MA、MACD、RSI、PE TTM、资金流、投行一致预期等指标优先直接获取权威数据源披露值；量比只在收盘口径下使用 `seed_finance_search（同花顺数据库）` 返回的当日成交量和过去 5 个交易日成交量自行计算，盘中口径统一写 `暂无`。若确需自行计算其他字段，必须写明公式、输入来源和计算过程，并在报告中标注为“自行计算”。
- 新闻、公告和研报必须核查原文真实性、发布日期和是否落在要求时间范围内。正文引用的重要事实必须有可打开来源；无法核验的传闻不得写成事实。
- 任一非强制专用字段若通过 `seed_finance_search` 未取得明确结果，且通过 `general_search` 也找不到官方或权威机构来源，可以留空并跳过对应展示或分析；不得为了完整性牺牲准确性。摘要指标 8 项和 A 股资金流向指标若 `seed_finance_search` 未返回，不得改用 `general_search` 补数，必须重试 `seed_finance_search` 或写 `暂无` / 暂停生成；港股/美股资金净流入/净流出和港股通南向资金可在 seed 未返回时用 `general_search` 查找官方或权威机构来源，但只接受报告当日或最近一个有效交易日的数据。

## 总工作流

### 1. 识别股票与用户问题

先同时识别两件事：

- 股票主体：股票名称、证券代码、市场、币种。
- 用户问题：每日复盘、目标价/上涨空间、短线交易、事件催化、持仓处理、风险判断、估值贵不贵等。

必须把用户问题转化成 `question_focus`，并贯穿对话版日报和后续飞书文档版正文。不要只生成模板化日报而不回答用户原问题。

常见 `question_focus`：

- `target_price`：用户关心股价能到哪里。重点看技术压力位、震荡区间、投行一致预期、估值分位、业绩弹性，给出情景区间而非确定目标价。
- `short_trade`：用户想短线快进快出。重点看今日量价、资金流、技术指标来源值、支撑压力和波动风险；只能给研究视角的短线环境判断，不给具体买卖、仓位、止损或止盈指令。
- `event_catalyst`：用户问销量、订单、公告、政策、行业新闻是否会拉动股价。重点核查事件真实性、收入/利润贡献、持续性、是否已被股价反映，并说明对短期情绪和中期基本面的不同影响。
- `position_advice`：用户提到持仓、空仓、被套、盈利。重点分持仓状态给出操作边界和风控。
- `daily_recap`：用户只要求复盘。按标准日报生成，但仍要提炼最重要矛盾。

如果用户问题不清楚，先用最接近的 `question_focus` 推断；只有股票主体无法确定时才向用户确认。

- A股：支持 `300308`、`SZ300308`、`300308.SZ`、`中际旭创` 等。
- 港股：支持 `00700`、`0700.HK`、`腾讯控股` 等。
- 美股：支持 `NVDA`、`MSFT` 等。

如果名称可能对应多只股票，先用搜索或行情接口确认；无法确定时向用户确认。

若同一公司同时有 A 股和港股上市主体，且用户只给公司简称、没有明确代码或市场，例如“紫金矿业”“中国平安”“比亚迪”，必须先询问用户要分析 A 股还是港股；在用户确认前不得开始取数和生成日报。若用户同时给出 A/H 两个代码但没有说明主分析对象，也必须先让用户选择“以 A 股为主 / 以港股为主 / 做 A/H 对比”，本 skill 默认只分析一个主交易标的。

### 2. 获取数据

必须联网或调用可用行情/数据工具，不要凭记忆。数据要求见 [references/data-contract.md](references/data-contract.md)，数据源优先级见 [references/data-sources.md](references/data-sources.md)。

在豆包等支持搜索工具的环境中，取数和资料搜集主要调用 `seed_finance_search` 与 `general_search`：`seed_finance_search` 是金融领域数据查询的第一入口，优先用于证券识别、行情摘要、PE TTM、成交量、A 股资金、板块、技术指标、预测、财务、汇率、基金、期货、贵金属和宏观数据；`general_search` 主要用于非查数类需求，例如公告、新闻、产业信息、公司介绍、分析材料和权威来源原文，也用于非强制专用数据在 `seed_finance_search` 搜不到结果时的数据兜底。摘要指标 8 项和 A 股资金流向指标只能查 `seed_finance_search（同花顺数据库）` 或基于其返回值自行计算，不得由 `general_search` 兜底；其中量比只在收盘口径下用 seed 返回的成交量自行计算，盘中写 `暂无`。港股/美股资金净流入/净流出及港股通南向资金不强制限定 seed，seed 未返回时可用 `general_search` 查找官方或权威机构来源，但数据日期必须是报告当日或最近一个有效交易日。任何经 `general_search` 采用的数据或事实，也必须来自官方或权威机构来源，不得采用雪球、股吧、贴吧、个人用户评论或无法确认原始来源的内容。

最低数据集：

- 当前价/涨跌幅/日内区间/总市值/PE TTM/成交额/换手率，且只能来自 `seed_finance_search（同花顺数据库）`；量比收盘后使用 `seed_finance_search（同花顺数据库）` 返回的当日成交量和过去 5 个交易日成交量自行计算，盘中写 `暂无`。
- A 股：主力资金、超大单资金、大单资金；陆港通标的追加北向资金，且只能来自 `seed_finance_search（同花顺数据库）`。港股：资金净流入/净流出；港股通标的可追加南向资金净流入/净流出。美股：资金净流入/净流出。港股/美股资金数据先查 `seed_finance_search`，未返回时可用 `general_search` 检索官方或权威机构来源，但数据日期必须是报告当日或最近一个有效交易日。
- 相关板块表现。
- MA、MACD、RSI、震荡区间、压力位、支撑位。
- 最近 3 日内权威财经新闻、产业新闻、公告或监管披露。
- 投行一致预期与短期风险。

按 `question_focus` 补充取数：

- `target_price`：补充一致预期 EPS/PE、历史估值区间、可核验压力位/支撑位、上方缺口/前高/整数位、可比公司估值。
- `short_trade`：补充当日快照、收盘位置、成交额相对 5 日均值、短线资金流和可核验关键波动区间，用于说明风险环境，不用于生成具体交易指令。
- `event_catalyst`：补充事件公告/新闻原文、事件发生日期、市场是否已交易该事件、对收入/利润/订单/产能/政策预期的可能影响。
- `position_advice`：补充成本区间无法获知时，用当前价上下关键位给出通用持仓/空仓/短线三类建议。

### 3. 格式交付原则

- 先生成完整 Markdown 内容底稿；该底稿是后续聊天渲染结果和飞书文档版的事实、观点、章节和来源编号基准。
- 聊天渲染结果只能在完整 Markdown 内容底稿独立完成后做展示层调优，不删减、不压缩、不重写内容；具体标题、空行、重点加粗、表格居中、推荐结构和展示检查见 [references/dialogue-style.md](references/dialogue-style.md)。
- 飞书文档版只基于同一 Markdown 底稿做 `lark-doc` 结构化和样式调优，不重新生成事实和观点；具体 XML、表格、颜色、列宽和渲染检查见 [references/feishu-doc-style.md](references/feishu-doc-style.md)。
- 飞书文档版核心格式必须保留：只使用 `lark-doc` 支持的静态标签；首屏为文档标题、短免责声明提示、元信息/公司简介、核心结论、摘要指标表、第一部分正文；主章节顺序与 Markdown 底稿一致；摘要指标、资金流、技术指标、关键价格区间、投行一致预期使用表格；文末必须有信息来源和免责声明。
- 飞书文档版中的摘要指标、资金流、技术指标、关键价格区间、投行一致预期必须使用 `lark-doc` 支持的 `<table>`、`<colgroup>`、`<th/td background-color>` 结构写入；缺失字段统一写 `暂无`。

### 4. 输出与飞书文档

- 完成数据获取和分析后，默认先输出对话版完整日报，不直接生成飞书文档版完整报告。
- 完成取数/搜索与内部数据核验记录后、生成完整 Markdown 内容底稿前，必须读取 [references/content-rules.md](references/content-rules.md)；不得在取数前提前读取该文件。
- 完整 Markdown 内容底稿必须先独立完成；底稿完成后、输出聊天渲染结果前，再读取 [references/dialogue-style.md](references/dialogue-style.md)，只做展示层调优和展示检查。
- 对话版日报固定以 `如果你需要，我可以把以上内容写入一份飞书文档。` 收尾。
- 只有当用户明确表示“生成飞书文档版 / 做成飞书文档 / 需要完整报告 / 做成文档 / 继续生成”时，才进入飞书文档编辑流程。
- 生成飞书文档版时，必须在组装 `lark-doc` XML 前从头到尾完整读取 [references/feishu-doc-style.md](references/feishu-doc-style.md)，并调用 `lark-doc` skill 新建或编辑飞书文档；不得只搜索关键词、只读取局部片段、只依赖本文件摘要，也不得只在聊天界面输出完整飞书文档正文来代替写入。
- 若当前环境无法写入飞书文档，必须明确说明阻塞原因，并询问用户是否接受先输出标签版草稿；只有用户明确同意时，才在聊天界面输出完整文档内容。

### 5. 飞书文档版交付复核

生成飞书文档版后不得立即交付。必须执行：

1. 全量复核数据、内容、飞书文档结构、lark-doc 样式、表格宽度、信息来源和免责声明。
2. 发现任意失败项，先修复，再从头全量复核。
3. 至少完成两轮复核。
4. 必须确认输出只包含 `lark-doc` 支持的标签和文本。
5. 必须优先通过 `lark-doc` 写入飞书文档；若可操作 Chrome/飞书页面，应打开文档检查真实渲染。若不能调用 `lark-doc`，不得直接输出完整正文，必须说明阻塞原因并询问用户是否接受标签版草稿。

复核细则见 [references/qa-checklist.md](references/qa-checklist.md)。

## 合规口径

本 skill 生成研究讨论材料，不提供保证收益或确定性买卖结论。必须保留免责声明：
