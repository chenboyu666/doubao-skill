## 聊天渲染器职责

请务必独立生成完整 Markdown 内容底稿后，再阅读本文档。本文件只规定完整 Markdown 内容底稿在聊天界面中的渲染方式。本文档的段落风格不应干扰 Markdown 内容底稿的风格。本文档不应该改变底层 Markdown 的事实、观点、章节、表格、来源编号和免责声明；后续飞书文档版必须与该底稿保持事实、观点和来源对齐。

## 聊天渲染原则：以下要求是为了保持内容完整同时满足读者扫读的需求

- 以底层 Markdown 为唯一内容来源，不新增或删除一级章节、二级小标题和信息点，不改写结论或逻辑顺序。
- 只优化聊天可读性，而非保持文档排版；使用 Markdown 输出，不使用卡片、emoji、彩色背景、渐变、夸张营销标题或多余分割线。
- 每段通常不超过 3 句话，每段要对呈现核心观点/判断/主旨的相应分句加粗，方便读者扫读。如果段落太长，可以拆分。
- 小标题与正文之间、章节内连续两段之间、章节之间必须保留空行，方便快速浏览。

## 标题与段落

- 对话版最顶部固定输出：`「以下内容仅为行情与逻辑分析，不构成任何投资买卖建议，务必结合自身风险承受能力决策」`。该提示语必须位于主标题之前，且全文只出现一次。
- 主标题固定为 `**个股日报 / 对话版深度复盘**`。
- 主标题下方固定一行元信息：`股票简称 证券代码 · 生成日期 · 研究讨论，不构成投资建议`。
- 一级章节标题固定使用中文编号和加粗，例如 `**一、今日行情、量能与资金流向**`。
- 小标题固定使用行内加粗冒号，例如 `**价格与量能：**`、`**资金流向：**`、`**研究判断：**`；不要新增另一套三级标题。
- `核心结论` 用 3-5 条编号列表，每条以加粗主题词开头，例如 `1. **直接回答：** ...`；第一条必须直击用户问题。

## 表格

- 聊天渲染中的 Markdown 表格只在资金流、技术指标、投行一致预期等确有结构化必要时使用。
- 表头名称与飞书文档版报告一致，不临时扩列或换名。
- 所有 Markdown 表格的分隔行必须使用居中格式：`|:---:|:---:|:---:|`。四列、五列表格依此类推，所有列都用 `:---:`。
- 表格前后各保留一个空行。

## 发送前展示检查

输出聊天渲染结果前，必须先做一次展示检查并修正：

- 一级章节之间是否保留空行。
- 段落之间是否保留空行。
- 是否对每段核心观点、重要事实做行内加粗。
- 表格分隔行是否全部使用 `:---:` 居中格式。
- 是否存在整段加粗。
