# 豆包舆情追踪与溯源

## 触发范围限制（非常重要！！！）

1. 仅在网页端和电脑端场景中触发，可以继续按本 Skill 执行。
2. 当用户使用手机端（即所处平台是手机端）时，不要触发本 Skill，不要进入本 Skill 的五步舆情工作流。

### 手机客户端场景下的处理方式

1. 简短提示用户：本任务使用电脑版或网页端效果更佳。
2. 不要求用户立刻切换设备。
3. 不使用本 Skill。
4. 继续根据用户当前请求，用模型自身能力完成能完成的部分，例如解释需求、整理已有信息、给出检索关键词、设计调研方案、说明下一步操作建议等。

推荐提示语：

“本任务使用电脑版或网页端效果更佳，现在会继续帮你完成任务。”

## 强制红线规则（违反任意一条视为不合格，必须返工）（非常重要！！！）

1. **每条舆情必须携带可直接访问的原帖链接（非常重要！！！）** —— 没有链接的内容一律不能输出，宁可少也不能凑数。只要有任何一条舆情缺少原帖链接，整个报告都算不合格。如果需要登录的，调用interaction.request_action，类型是browserControl，让用户完成登录。绝对禁止更换无需登录的平台、禁止关闭弹窗，禁止同时去搜别的平台，也禁止认为收集到的资料已足够而轻易放弃。必须让用户帮助完成登录
2. **优先使用一手信源** —— 用户直接发布的原始帖子 > 媒体报道 > 二手转述
3. **二手信源必须明确标注原因** —— 只有在穷尽所有方法确实找不到原帖时，才能使用二手信源，且必须说明"为什么找不到原帖"
4. **不允许OrganizeAgent参与**
5. **必须使用文档交付结果，包括一手信源链接；除非用户明确指定其他格式（非常重要！！！）**

## 技能概述
用于搜索、筛选、验证和整理公开舆情，进行全网舆情搜索与内容溯源，采用"全面检索+深度挖掘"的双层策略，确保舆情覆盖全面，并且所有找到的舆情内容都能追溯原始发布者、输出可点击的一手信源，并清楚说明无法覆盖的平台或只能使用二手信源的原因。

## 核心原则
### 1. 全面覆盖第一
舆情检索首先要确保覆盖面广，不能局限于少数几个平台。先通过全网检索获取广泛信息，再针对重点平台进行深度挖掘。

### 2. 溯源第一
**每条舆情必须携带可直接访问的原帖链接**，所有找到的舆情内容必须提供原始帖子的直接链接和原始标题，不允许只转述内容而不给出处。发现舆情内容后，必须追溯到其原始发布平台和原帖。

### 3. 一手信源优先
优先使用用户直接发布的原始帖子作为信源。媒体综合报道、洗稿文章、二手转述等都不是一手信源，除非确实找不到原始帖子，否则尽量不用。

### 4. 质量优先
优先收集高互动量、有实质内容、有代表性的舆情内容，而非追求数量。

### 5. 全面覆盖不缩水
即使用户明确提到了某一个或某几个特定平台，也**不能只调研用户提到的平台**。必须在用户指定平台的基础上，额外完成本技能规定的其他核心平台的调研，确保舆情覆盖的全面性。用户提到的平台只是"重点关注"，不是"只看这个"。

## 工作流程（五步法）

### 第一步：全网检索（确保全面性）
使用通用搜索工具进行全网检索，获取广泛的舆情信息。
- 搜索关键词：根据用户需求确定核心关键词，通过搜索寻找同名近义词，如对应英文名称、社区常见昵称等
- 覆盖范围：新闻网站、自媒体、社区论坛、社交媒体等
- 目的：快速了解舆情整体态势，发现有价值的线索

**注意**：通用搜索结果找到的可能为二手信息（新闻报道、综合文章等），不能直接作为最终信源，需要进一步溯源。

### 第二步：重点平台深度检索
在全网检索的基础上，针对重点社交媒体平台进行深度检索和使用浏览器工具浏览（必做）
浏览器工作可以通过/gui-browser-task-skill或/browser-task 这个skills来使用

**⚠️ 重要提醒：即使用户指定了某个平台，也不能只做那个平台。用户指定的平台只是"重点关注"，其他核心平台也必须覆盖。**

**核心重点平台**（按优先级排序，必须全部覆盖）：
1. **微博** - 国内舆情主阵地，话题传播快，讨论量大
2. 其他用户明确要求的平台

**其他可能的重要平台**（根据舆情情况选择性检索）：
- 知乎 - 深度讨论、专业分析
- 即刻 - 互联网从业者讨论
- 脉脉 - 职场、行业相关讨论
- B站 - 视频形式的评测和讨论
- 豆瓣 - 小组讨论
- Twitter(X) - （如果用户不明确要求，则不检索此平台）海外舆情、英文讨论，注意搜索词需要用中文和英文分别尝试
- 其他用户喜爱的社区和垂直社区

**检索方式**：
- 直接访问平台网站进行搜索和浏览
- 查看热门帖子、高互动内容
- 浏览评论区获取更多用户真实反馈

### 第三步：内容筛选与初判
从全网检索和平台检索的结果中，筛选出有价值的舆情内容。
筛选标准：
- 高互动量（转发、评论、点赞数高）
- 有实质内容（不是简单灌水）
- 有争议或代表性观点
- 时间较近（优先最近3个月内）
- 发布者有一定影响力或代表性

### 第四步：强制溯源（最关键步骤）
对筛选出的舆情内容，必须追溯到原始发布平台和原帖链接。

#### 溯源场景与方法
**场景1：新闻报道/自媒体综合文章**
- 特征：标题通常是"XX被吐槽"、"XX引发热议"、"网友怎么看XX"等
- 性质：二手信息，不是一手信源
- 处理方法：
  1. 仔细阅读文章，找到提到的原始帖子线索（用户名、发布时间、内容片段等）
  2. 根据线索去对应的社交媒体平台搜索原始帖子
  3. 找到后复制原帖链接，作为最终信源
  4. 只有在以下全部条件满足时，才能使用二手信源：
  - 已尝试在对应平台搜索（用户名、关键词、时间范围）
  - 已尝试通过新闻内容反向溯源
  - 确认该帖子可能已被删除或设为私密
  - 且必须标注：信源类型：二手信源（因XXX原因未找到原始帖子）

**场景2：二手转述/截图传播**
- 特征：内容是"有人说..."、"看到一张图..."等转述形式
- 性质：二手信息，需要验证和溯源
- 处理方法：
  1. 根据内容特征判断可能的原始平台
  2. 去对应平台搜索找到原始发布者和原帖
  3. 验证内容的真实性和完整性
  4. 使用原始帖子作为信源

**场景3：直接在平台搜索到的帖子**
- 特征：直接在微博等平台搜索到的用户原始帖子
- 性质：一手信源
- 处理方法：
  1. 确认是用户直接发布的原始内容
  2. 复制每条帖子的直接链接
  3. 记录完整信息（发帖人、时间、内容、互动数据）

**场景4：多平台转载内容**
- 特征：同一内容在多个平台出现
- 处理方法：
  1. 判断哪个平台是原始发布平台
  2. 优先使用原始发布平台的链接
  3. 如无法判断，使用最早发布的版本

✅ 溯源后验证：
- [ ] 是否已获取到原帖的直接链接？


### 第五步：信息整理与输出（先总结，后详情）

**输出结构必须遵循：先归类总结 → 后详情展开**
**输出内容：必须使用文档**

#### 第一部分：舆情归类总结
先对所有收集到的舆情进行正负向归类和总结，让读者一眼看清整体态势：
- 正向舆情有几大类，每类的核心观点和代表性
- 负向舆情有几大类，每类的严重程度和热度
- 整体舆情态势判断（正面为主/负面为主/两极分化）

#### 第二部分：详细舆情列表
按平台或按主题分类展开具体每条舆情
优先选择用浏览器工具浏览过的帖子的内容和链接
每条必含：
- 发帖人/发布者
- 发布时间
- 内容摘要
- 原帖链接（原始发布平台的直接链接）
- 互动数据（转发/评论/点赞，如可获取）
- 信源类型标注（一手信源/二手信源）

## 使用说明

1. 收到舆情搜索需求时，自动触发本技能
2. 按五步法执行：全网检索 → 重点平台深度检索 → 内容筛选 → 强制溯源 → 整理输出
3. 优先使用一手信源，二手信源需标注原因
4. 如遇到平台登录限制，请求用户协助登录
5. **输出结构必须遵循：先正负向归类总结，后详细列表**
6. **每条舆情必须携带原帖链接，缺一条都算不合格**
7. **即使用户指定了某个平台，也必须覆盖所有核心平台（微博等），不能只做用户提到的那一个**
8. **默认使用豆包文档输出结果，如果用户要求直接输出/ Markdown 输出，则直接用 Markdown 输出。**
9. **客户端限制**：本 Skill 仅在用户使用网页端或电脑客户端时触发；如果用户使用手机客户端，不触发本 Skill，只提示“电脑版或网页端效果更佳”，然后由模型继续完成用户任务。
