# 报告结构：必写内容、视情况写的内容与降级规则

## 原理

结构服务于两件事：让读者 30 秒内看到本季的核心信息，让关键证据数字都能追溯到原始材料。结构不是配额——**篇幅跟随展开的交叉检验数量**，展开几组就写几组，每组推到深度3。任何一段如果删掉后不影响读者对主线的理解，就应该删掉。

## 标题规则

事实性结论标题：数字 + 方向 + 本季核心特征，禁止立场词和戏剧化措辞。

- 合格：`投入重置利润表：经调整 EBITA 降 84%、云收入加速至 38%，市场定价重心转向投入回报的兑现`
- 违规：`豪赌 AI！利润主动清零换增长`（立场词）；`XX 公司 2026Q1 财报分析`（无信息量）

## 输出标题层级

最终报告的 markdown 标题层级固定如下：

```
# [报告标题（事实性结论标题）]
## 本季概览
## 业绩摘要
## 预期差拆解
## 核心指标解读
### 1. [维度 A]
### 2. [维度 B]
### ...
## 管理层信号表
## 机构观点汇总（视情况）
## 后续关注清单
## 数据来源
### 风险提示与免责声明
```

- `#` = 报告标题（全文唯一）
- `##` = 主体章节
- `###` = 仅用于「核心指标解读」下的各论证维度（必须带阿拉伯数字序号：`### 1.`、`### 2.`…）以及文末「风险提示与免责声明」

## 必写内容（每篇必有，顺序如下）

### 1. 标题区

公司全称、代码、报告期、报告日期。**不含评级**——本报告不输出评级与目标价。

### 2. 本季概览

一段话或 3-5 个要点：本季核心特征（通常是一组反差或一个主导变量）+ 报告围绕哪几个问题展开。若有机构行为可佐证市场解读重心（如「财报后机构上调的是 X 预期而非 Y 预期」），具名转引。

### 3. 业绩摘要表

一句话判定超预期/符合预期/不达预期（预期锚需注明来源机构和日期，且日期须早于财报发布日）+ 摘要表：实际、预期锚、偏离、同比、环比。表下必须有来源行和**口径提示**（如环比含淡旺季、某基数含非经常项）。无一致预期时删除锚列并声明「未获取」。

### 4. 预期差拆解

不报幅度报结构：偏离里多少是需求、多少是会计/一次性、多少是成本与投入节奏。收入与利润偏离的不对称本身是信息。

**锚点-措辞绑定**：「超/低于预期」只能用于有明确预期锚的指标（绑定关系见 analysis-moves.md 动作二）；无锚指标用同比/趋势措辞。无任何锚时整节降级为 vs 历史趋势/公司指引，并声明。

### 5. 核心指标解读

主体章节。按 `_INTERNAL_DO_NOT_DELIVER__READ_00_RESUME_FIRST/DO_NOT_DELIVER__writing-plan.md` 的筛选结果：展开的交叉检验各成一小节（写法见 analysis-moves.md 动作三）；一句话通过的检验项合并为一段「表现合理指标项」；如触发缺口测算，相关内容也放在本节。收入主拆分按命中原型量纲做，分解项复合增速 ≈ 收入增速（±5pct）。

**格式要求：** 本节下的每个论证维度使用 `### + 阿拉伯数字序号` 作为小节标题（如 `### 1. 为什么需求超出市场估计`、`### 2. 毛利率为什么能连续六个季度提升`）。序号连续编排，帮助读者快速定位和跨节引用。

### 6. 管理层信号表

三栏表：信号原文要点 / 数字含义（定性表述翻译成可检验数字）/ 风险与后续关注。不评判对错。

降级：本期无说明会 → 用公告内展望性表述，写明「本期无业绩说明会」；完全无表态 → 本节一句话声明，关注点顺延。

### 7. 后续关注清单

3-6 条，按信息量排序，每条注明用什么公开数据验证。这是「方向指引」的固定交付物，**不依赖机构观点存在**——它从交叉检验（每个展开的检验项天然产出一条）和信号表（每条信号的关注列）汇总而来。本季未裁决的关注点必须出现在清单中。

### 8. 数据来源

- 来源分组：公司公告及定期报告 / 说明会与新闻 / 机构研究报告 / 行情数据，每条带名称、日期、**可点击链接**（链接从 `_INTERNAL_DO_NOT_DELIVER__READ_00_RESUME_FIRST/DO_NOT_DELIVER__facts.json` 的 url 字段来；无公开链接的写平台名）。
- **未获取清单**：按对分析的影响排序，写明缺什么、影响哪段分析。用读者语言，不提内部文件。
- 免责声明：声明所引机构观点不代表本报告立场，不构成投资建议。

### 风险提示与免责声明：

## 视情况写的内容

### 机构观点汇总（位于关注清单之前）

- **≥2 家具名财报后观点** → 单独成章：机构观点表（机构/观点/日期与转引出处）+ 一段「市场分歧在哪里」（用客观事实呈现分歧，如一致目标价区间宽度，不替任何一方站队）。
- **仅 1 家** → 不成章，并入相关段落转引。
- **0 家** → 在未获取清单写明「财报后机构观点未获取」。

机构的评级与目标价只能作为本表的转引条目出现，不得据此推导「上行空间」之类的自有结论。

### 缺口测算

仅当存在重大未披露量（投入总额、分部亏损、产能、单价），且可由已有数字从独立通道倒算时使用，写法见 analysis-moves.md 动作四。

### 图表

仅当图比文字承载更多信息时插入（如 5 季以上趋势、结构对比）；数据只来自 `_INTERNAL_DO_NOT_DELIVER__READ_00_RESUME_FIRST/DO_NOT_DELIVER__facts.json`，每图有来源行。

## 引用规范（读者可见）

源稿是面向最终读者的成品。一切措辞、标注、来源说明都必须是读者可直接理解的语言。内部流程词（`_INTERNAL_DO_NOT_DELIVER__READ_00_RESUME_FIRST/`、facts.json、writing-plan、复算、门禁等）一律不得出现。

### 引用方式：源稿绑定事实，finalize 自动生成来源标记

源稿不要手写 `[1]`、`[^1]`、`[^2][^3]` 这类来源角标。源稿只写 `{fact:claim_id}` 绑定；`finalize_report.py` 生成 display markdown 时，会把绑定自动转换成普通文本 `[n]` 标记，并按同一编号在文末「数据来源」列出对应来源。

`{fact:claim_id}` 不是数值占位符，也不会把 claim 内容自动填回正文。正文必须先写出读者可见的完整数字或判断，再把 `{fact:...}` 放在该数字、表格行或句子之后。禁止写 `自由现金流为{fact:fcf_2026}`、`资本开支达{fact:capex_2026}` 或表格单元格只写 `{fact:capex_2026}`；正确写法是 `自由现金流由 2025 财年的净流入 738.70 亿元转为 2026 财年的净流出 X 亿元。{fact:fcf_2026,fcf_2025}`。

范例：`营业收入 195.0 亿元，同比 +192%，超一致预期约 +44%。`

### 源稿事实绑定：`{fact:claim_id}`

源 markdown 建议在支撑结论的关键数字、表格关键行和关键事实判断后绑定 `_INTERNAL_DO_NOT_DELIVER__READ_00_RESUME_FIRST/DO_NOT_DELIVER__facts.json` 的 `claims`，用于生成来源标记，并让 lint 复核 `usage_type` 与正文语气是否匹配：

`国外销售占比从 2024 年的 53.49% 降至 2025 年的 49.44%。{fact:overseas_share_2024,overseas_share_2025}`

这些 `{fact:...}` 绑定只存在于源稿，`finalize_report.py` 生成 display markdown 时会自动转换成普通文本 `[n]` 来源标记。对话输出不得出现 `{fact:...}`，源稿不得手写 `[n]` 或 `[^n]` 这类角标。缺少绑定、来源语气疑似不匹配时，lint 输出警告，由模型复核后决定是否修改。若 `{fact:...}` 被放在“为/达/至/实现/录得”等谓语后直接充当数字，lint 会报错；这说明正文漏写了事实值，需要先补正文数字再绑定。

### 派生数字

由公开披露数据计算得出的数字，正文只写结论，不暴露中间算式：

- 正确：`超一致预期约 +44%`
- 正确：`预付款较年初增长约 10 倍`
- 错误：`超一致预期约 +44.2%（194.96÷135.2−1）`
- 错误：`（测算：1,964.58÷1,980−1）`

### 文末来源表

标题为「文中引用对应以下来源：」，每条格式为：`编号. [材料名称](链接)`。

- 公司公告、研报等有公开 PDF/网页链接的，写成 markdown 超链接（飞书文档应保留为可点击链接）。
- 媒体转引无直接链接的，写明媒体名+日期。
- 专业数据库（Wind/Choice 等）无公开链接的，写平台名+日期。

### 未获取清单

在来源表之后，格式为 `- **未获取清单**（按对分析的影响排序）：- …`。用读者语言写明缺什么、影响哪段分析。

### 禁止出现的内部流程词

`_INTERNAL_DO_NOT_DELIVER__READ_00_RESUME_FIRST/`、facts.json、writing-plan、company-brief、analysis-brief、reader-outline、report-source、analysis-source、DO_NOT_DELIVER__、FINAL_REPLY_BODY、00_RESUME_HERE、复算、门禁、check_facts、lint_report、make_charts——这些是作者侧工具，不是读者应看到的内容。`lint_report.py` 会扫描并报错。

## 降级规则总表

| 缺失 | 处理 |
|---|---|
| 一致预期 | 摘要表删锚列；预期差拆解改用历史趋势/公司指引，声明「未获取一致预期」 |
| 业绩说明会 | 信号表降级为公告展望表述，写明 |
| 财报后机构观点 | 机构观点不成章，写入未获取清单；关注清单照常 |
| 上季/历史数据 | 环比与趋势分析省略，声明 |
| 分部数据（A 股季报常态） | 收入拆分降级为披露口径能做到的层级，写明披露限制 |
| 行情数据 | 不影响交付（本报告不做估值）；机构观点表照常转引 |
