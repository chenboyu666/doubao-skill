# 分析动作手册（报告模式专用）

阶段 2（写作决策）与阶段 3（写作）必读。深度不来自篇幅和形容词，来自六个动作。每个动作都有「什么时候不做」的规则——决定不做和决定做同样重要。

推理质量标准（解释深度阶梯、推理三步循环、结论分档、锚点-措辞绑定）见 `reasoning-framework.md`，报告模式同样适用。本文件聚焦报告模式的写作流程与结构化动作。

## 目录

- 动作四到六：缺口测算、管理层信号翻译、关注清单汇总
- 语气三句式与交稿前自检

## 写作决策模板（`_INTERNAL_DO_NOT_DELIVER__READ_00_RESUME_FIRST/DO_NOT_DELIVER__writing-plan.md`）

动笔前填完，10-20 行：

```markdown
# 写作决策

- 主线：本季唯一核心特征一句话（通常是一组反差或一个主导变量）
- 锚点清单（逐指标，无锚的正文禁用「超/低于预期」措辞）：
  - 收入：有锚（____，日期）/ 无锚
  - 归母净利润：有锚 / 无锚
  - EPS：有锚 / 无锚
  - 毛利率及原型核心 KPI：有锚 / 无锚
- 关注点裁决状态（来自 `_INTERNAL_DO_NOT_DELIVER__READ_00_RESUME_FIRST/DO_NOT_DELIVER__company-brief.md` 的 3 个关注点）：
  1. ____：已裁决/部分裁决/未裁决（裁决数字：____）
  2. ____：…
  3. ____：…
- 交叉检验筛选（按命中 playbook 清单逐项过）：
  - 检验项 A：展开（过门槛：____）/ 一句话通过 / 不适用
  - 检验项 B：…
- 证据缺口清单（主线与每个拟展开检验项依赖的关键证据）：
  - 证据 1：已有 / 缺失但可得（检索方向：____）/ 缺失且不可得
  - 证据 2：…
  （「缺失但可得」项进入阶段 3.5 定向补查，每缺口 ≤2 轮；
   补查后仍缺的，对应结论按「结论分档」降档）
- 缺口测算：触发/不触发（未披露量：____；可用通道：____）
- 机构观点：成章（N 家具名）/ 并入段落（1 家）/ 未获取
- 图表：需要哪几张 / 不需要
```

**篇幅原则**：篇幅跟随展开的交叉检验数量，不设配额。展开几组就写几组，每组推到深度3。没有值得展开的检验项 = 本季无重大新信息，直接告诉用户「本季无增量信息，建议关注 X/Y/Z 在下个窗口验证」，不出报告。

## 动作一：主线判定

每季财报只有一个核心特征，全文篇幅向它倾斜。主线从两处找：预期差的结构（哪里和市场想的不一样）+ `_INTERNAL_DO_NOT_DELIVER__READ_00_RESUME_FIRST/DO_NOT_DELIVER__company-brief.md` 关注点（市场本来在等什么答案）。次要分部一笔带过或进「校验通过清单」，不平均用力。

判断一段话去留的标准：删掉它，读者对主线的理解受损吗？不受损就删。

## 动作二：预期差拆解

预期差拆解重结构、轻幅度。把偏离拆到三类：需求端、会计与一次性、成本与投入节奏。

> 范例（结构拆解）：收入低于预期 1.3%，经调整利润低于预期约 99%——不对称本身就是结论：偏离几乎全部来自投入节奏而非需求。收入端缺口的相当部分是会计因素（补贴重分类为收入冲减拖累披露口径 7pct）；利润端缺口来自投入规模超出卖方模型；GAAP 净利 +106% 为非经常项，不进入任何同比结论。

全部无锚时降级：vs 历史趋势或公司指引，声明「未获取一致预期」。

## 动作三：数字交叉检验（深度的主要来源）

把「放在一起不舒服」的数字成对检查。**清单在命中 playbook 的「常见检验项」小节**——按清单逐项过，不靠灵感，也不许超出清单硬造。

每一组有三种处理方式：

1. **展开**（成段）：必须过重大性门槛（playbook 里每项都有具体门槛，通用兜底：偏离 >5pct / 口径差 >3pct / 两数字方向相反 / 连续多季趋势拐点 / 数字与管理层表态矛盾）。
2. **一句话通过**：`量价拆分无异常：量 +4%、价 +2%，与收入 +6.1% 吻合`。告诉读者「这里查过了，没事」是分析的一半。
3. **不适用**：该原型/该公司无此量纲，跳过不提。

展开的固定写法：**现象（两个数字并置）→ 可能解释（含公司口径，不止一种）→ 后续应关注（用什么公开数据验证）**。

从「现象」到「可能解释」的推理路径，须经过 `reasoning-framework.md` 的三步循环（分解→假设→自检），**展开的检验底线是深度3**——必须回答「为什么公司不能消解这个变化」（竞争格局、供应链位置、客户议价权等约束），读者才能判断这件事的严重性和持续性。停在深度2 = 你只说了「发生了什么」，没说「为什么公司改变不了它」。

> 范例（展开）：即时零售收入 +57%，订单量为去年同期 2.7 倍，隐含单均收入同比约 -42%。可能解释包括订单结构向低客单价品类倾斜，或补贴以收入冲减形式压低单均确认收入——公司未披露拆分。管理层称 4 月以来「高笔单价占比提升、UE 显著改善」。后续应关注：「收入增速÷订单增速」剪刀差是否收敛。

## 动作四：缺口测算（条件动作）

当存在重大未披露量（投入总额、分部亏损、单价、产能）时，用底稿数字从 1-2 个独立通道倒算量级。正文只写结论数字和量级，不暴露中间算式；标注口径局限、说明通道之间能否加总。

> 范例：公司未披露资本开支。现金通道测算单季资本性支出约 267 亿元；损益通道测算单季利润让渡约 268 亿元。两通道性质不同不可加总。

不触发时不要硬造——没有重大未披露量就跳过本动作。

## 动作五：管理层信号翻译

三栏表：**信号原文要点 / 数字含义 / 风险与后续关注**。两条原则：

1. 定性表述必须翻译成可检验的数字。「占比一年内 30%→50%」→ 推算对应的绝对额和所需增速。
2. 不评判目标能否实现。合法动作是：给出兑现路径的数字含义 + 指出哪个公开数据能提前观察到兑现或跳票。

> 范例行：`即时零售 UE 2027 财年末转正 ｜ 管理层主动给出带时间点的可检验承诺 ｜ 验证手段：收入÷订单剪刀差`

无说明会时本动作降级（见 report-structure 降级表）。

## 动作六：关注清单汇总

报告固定交付物，3-6 条，按信息量排序。来源有三：每个展开检验项的「后续应关注」、信号表的关注列、本季未裁决的关注点。每条注明验证数据来源（公司披露项 / 竞品财报 / 行业数据）。

## 语气三句式：正反例对照

| 违规写法 | 合规改写 |
|---|---|
| 我们认为云业务的增长可持续 | 云收入连续 N 季加速，AI 占比 30%（业绩公告）；高盛将云增速预期上调至 23%/25%（具名转引）；后续应关注增速能否站住 35% |
| 利润下滑但我们预计两个季度内见底 | 市场一致预期经调整 EBITDA 于 6 月季度回升至约 350 亿元（TIKR 2026-06-02）；后续应关注下季环比方向 |
| 公司估值已具吸引力 | （删除。本报告不输出估值判断） |
| 该目标大概率无法实现 | 该目标隐含五年 CAGR 约 35%，对照当前增速 38%；后续应关注逐年增速能否站住 35% |
| 业绩低于预期，前景堪忧 | 偏离主要来自 X 而非 Y；风险与验证指标见关注清单第 N 条 |

## 交稿前自检（阶段 4，只查脚本管不了的事）

1. 正文核心证据数字与 `_INTERNAL_DO_NOT_DELIVER__READ_00_RESUME_FIRST/DO_NOT_DELIVER__facts.json` / check_facts 复算输出核对一致，并优先在源稿中绑定 `{fact:claim_id}`；lint 提示的未绑定、期间、来源语气警告必须逐条复核。
2. 每个「展开」是否真过门槛；是否存在删掉后不影响主线理解的段落。
3. 三个关注点是否都交代了裁决状态；关注清单是否覆盖未裁决项。
4. 是否存在删掉后不影响主线理解的段落（有就删）。
5. 每个「超/低于预期」措辞对应的指标在锚点清单里有锚；主线因果链的结论分档与证据等级一致。
6. 证据缺口清单中「缺失但可得」的条目是否都做过补查；补查失败的结论是否已降档。
